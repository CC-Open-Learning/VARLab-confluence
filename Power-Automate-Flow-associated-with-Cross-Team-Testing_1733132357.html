VARLab : Power Automate Flow associated with Cross Team Testing

Created by Neethu Baby on Apr 03, 2025

Pilot - Automatic Assignments for Cross Team Testing is the power automate used for the automatic assignment. It is a manually triggered workflow, allowing us to initiate it based on the sprint.

Implementation Step

  1. Getting all items from the CTT Assignments list.

  2. Delete the old assignments.

  3. Get all items from the CTT Testers list.

  4. Set the column value “Assigned” as No.

  5. Get all items from the CTT Projects list.

  6. Loop through each item in the CTT Projects list. Check if the column value “Requesters Testers” greater than 0.

  7. If No,

    1. Send an email to Cam and Jon, saying no testers requested for this project

  8. If Yes,

    1. Get all items from CTT Testers list

    2. Loop through each item.

    3. Check is column value “Assigned” is not true AND column value “Requested Tester” greater than 0 AND column value “Availability for testing” is true

    4. If No,

      1. No action required

    5. Yes,

      1. Check if the column value “Primary Project Team” in CTT Testers list is not equal to the column value “Project” in the CTT Projects list.

      2. If No,

        1. No action required

      3. If Yes,

        1. Update column value “Assigned” as Yes in CTT Testers

        2. Create an entry in the CTT Assignments list

Note:

As we currently do not have a service account, the workflow is owned by Neethu and shared only with Jonathan and Dayne to minimize risks. Any incorrect changes to the workflow could potentially disrupt the entire process.

The "Title" column has been renamed to the "Project Name" column is in the CTT Projects list, and the "Name" column is in the CTT Testers list in SharePoint. In Power Automate, it will appear as "Title," which is the display name of the column.
