Overview

This page offers information about the Power Automate flows that support the cross-team testing process. These flows are required to capture MS Forms responses in the SharePoint list and send reminder emails.

Cross Team Testing - Record MS Form responses in SharePoint list - Power Automate Flow Steps

Whenever a user or tester submits their MS Form response, Cross Team Testing - Record MS Form responses in SharePoint list workflow will trigger and create an entry in the CTT Responses SharePoint list. This list serves as a repository for all cross-team testing results. We can filter the results by the submitted date column to view data specific to each sprint.

Currently, the "Cross-Team Play Testing: Work Progress Survey" MS Form is connected to a Power Automate flow. In the future, if a new form is created for cross-team testing, you must update the Form ID in the Power Automate flow to ensure that responses are added to the "CTT Responses" SharePoint list. Otherwise, the responses will not be recorded in the SharePoint list.

Cross Team Testing MS Form - Delete old responses and set a new end date - Power Automate Flow Steps

Cross Team Testing MS Form - Delete old responses and set a new end date workflow is scheduled to trigger biweekly on Thursdays, sending a reminder email to the respective individuals to set the end date for the MS Form and delete the old responses, effectively preparing the MS Form for the next sprint.

Note: As we currently do not have a service account, the workflow is owned by Neethu and shared only with Dayne to minimize risks. Any incorrect changes to the workflow could potentially disrupt the entire process.
