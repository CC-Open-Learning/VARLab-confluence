VARLab : Creating a 3D Asset

Created by Jordan Robinson, last modified on Aug 06, 2021

This article and its child articles are under construction.

In this article we will cover how to create a 3D Asset for CORE from modelling to importing.

  • High Poly Model
  • Low Poly Model
  • UV Mapping
  • Baking
  • Level of Detail Models

High Poly Model

Our first step is to create our high poly mesh, you may use Blender, Maya, 3DSMax, ZBrush, or any other 3D software for this step.

While our highpoly mesh is very pleasing to look at, it can be hundreds of thousands of triangles which will not perform very well in a game engine.

In order to transition our nice model into the game engine, we need to reduce its triangle count significantly.

Low Poly Model

In order for our model to work within the game engine, we will need to reduce the quality as much as possible without losing the overall silhouette of it or making it look too blocky and flat.

For this example we are reducing the detail of a highpoly mechanical model that was designed using the supporting edge-loop subdivision method. For any highpoly meshes made using a different workflow, such as through ZBrush using dynamesh, decimating and retopologizing for use within a game engine will be a much different process.

Before we begin reducing our highpoly, we’ll start by saving a copy to use later for baking. To start, we can remove the subdivision from our highpoly to give us the original mesh and its support loops.

Now we can remove the supporting edge loops and any loops that provide too much detail when compared to the rest of the mesh. A suggestion for this sort of workflow is to create a hotkey for your modelling program’s equivalent of the Select Loop, Select Ring, and Remove functions.

Now we can see the detail has been significantly reduced but we still have a lot of useless geometry in the middle here, cutting it out entirely and capping it to create a single quad will work best.

We can also see that when the rest of the model is visible, the portion we’re currently working on cannot be seen, so we are safe to delete it.

After repeating this process for the rest of the model, we’re left with a much simpler geometry.

Before we UV map we can finish off our lowpoly by fixing all the smoothing groups and setting them as we see fit. This top piece shows a lot of hard edges which we’ll fix with smoothing groups.

UV Mapping

With our lowpoly model ready, we can now think about doing our UV mapping. However looking at this model there are a few areas with large parts of the UV space dedicated to triangles that cannot be seen as they are covered by other parts of the model.

In order to use less UV space and thus have more pixels in our texture to dedicate to the visible parts of the model, we will increase our triangle count a bit.

Now with this large space opened up, when we go to UV map our object we will have significantly more space to work with.

After repeating this process for the rest of the model, we can see our triangle count is higher but not by much while our gains in UV space are significant.

Now with the model truly ready to go, we can begin UV Mapping. There are many ways to unwrap a mesh but we’ll focus on a simple one. First open the UV unwrapper with the entire model selected.

Find the edges of the model that you need to break in order to fold the entire thing open.

Break these edges and then hit Relax to have the computer automatically finish your UV for you.

Repeat this process for the rest of the object, combine them all together and open up the UV editor once again.

To give everything the same Texel Density, click your program’s equivalent of Rescale Elements to size each piece relative to the others based on their physical size.

Now with everything unwrapped and scaled properly, we just need to pack the UV as tight as we can.

With our UV tightly packed, it’s time to bake!

Baking

There’s a lot of software that can perform a highpoly to lowpoly bake for you, depending on the software you use this process will be different though the core ideas will remain the same.

First, reimport your highpoly on top of the lowpoly and add a Projection modifier in order to select the highpoly meshes you want to project onto the surface of your lowpoly.

Next you’ll want to increase the distance of the projection cage from your model in order to capture the details you’ve modelled.

With the cage size increased we can see it encloses most of the highpoly geometry but fails to catch the screws, for that we’ll have to manually increase the size of the cage for that area.

With the cage all set up for the model we can now begin baking our highpoly into normal and occlusion maps for our lowpoly. First you will need to change your render settings to have whatever lighting engine you’d prefer and also set the size and type of textures you would like to bake.

Make sure to specify some padding in your bake to prevent any texture bleeding later on.

Our final result looks like this, a nice lighting map and normals with a few intentional baking errors shown in red. To combat these errors, we simply have to modify the shape of our cage so that all of the highpoly is covered.

Level of Detail Models

Now that we’ve got a fully prepared lowpoly, we need to consider now if we’re going to need Level of Detail (LOD) models. These models are lower quality versions of our low poly meant to be viewed at further distances from the camera.

We may want this object to take up 1,500 triangles when viewed up close but when it’s 50m away it could probably be 100 without the user noticing. This increases memory costs a teeny bit for the user but significantly helps with performance.

We’ll start by making a copy of our model and add “_LOD1” to its name.

Now we can reduce its detail, keeping in mind this is the first level of reduction for this model’s detail. It can be a good idea at this time to apply the model texture or a checkerboard pattern so that you know if you accidentally break the UV.

With just a few edges removed, we’ve reduced the triangle count by nearly a quarter and it’s hard to tell from where. As we move further down our LODs the lack of detail will become more and more apparent.
