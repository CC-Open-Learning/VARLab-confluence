Value of Scriptwriting

In order to ensure a high quality of material to the learners, scriptwriters are included in the VARLab development team. The scriptwriter is responsible for ensuring written and spoken dialogue is:

  1. Understandable - is clear to learners from various backgrounds and education levels.

  2. Consistent - uses terminology correctly and consistently throughout.

  3. Inclusive - avoids assumptions about who the learners are.

Scriptwriting in the Development Process

This diagram shows the development process at a high level.

The scriptwriter is a member of the development team, with scripts being written at the same time as the code over a number of sprints. However, scriptwriters may be on multiple development teams at the same time, and will likely be moved to new teams once the script is drafted. This is because coding and art development are generally more time-intensive than scriptwriting. However, scriptwriters may revisit the project if the scope increases (e.g. additional development) or if there are significant changes (e.g. topic re-ordering, bug fixes).

Script Review Process for Voiceover Recording

Before a script can be sent for voiceover recording, it must be reviewed, using the following steps:

  1. Save script draft to project Sharepoint folder.

  2. Link script to Jira ticket and move ticket to Internal Review.

  3. Chat reviewer to trigger review.

  4. Reviewer documents issues and recommends changes using comments.

  5. Reviewer informs writer review is complete.

  6. Writer reviews and may or may not implement changes - changes not implemented MUST be discussed with the reviewer!*

  7. Jira ticket is moved to Done, and script can be sent through the appropriate channels (PI or Pedagogy Expert) to voiceover team.

* Why must changes not implemented be discussed? The reviewer wanted a change to fill a NEED. Discuss that reason and work together to ensure that need is met.

Scriptwriting Tips and Tricks

Script document

The script document is a document, spreadsheet, or Confluence page which has only the script and text used in the simulation. It will be used to populate the speech and message logic, and may also be shared with voice artists for recording.

  • Organize in order topics are presented

  • Identify caption by character if there are multiple characters

    • If necessary, describe characters for artists/voice artists

  • Identify caption by type of text - on screen information message, on screen prompt/list, spoken dialogue

  • Identify scenes/topics with colour coding or a searchable placeholder

  • Identify each caption with a unique string

  • Identify sections to be worked on or to be reviewed with colour coding or a searchable placeholder

  • Make sure captions are not so long that they cover the visual assets of the simulation

    • Each line is a caption

    • Unless the sentence is extremely short, usually 1 sentence/caption for reading ease

Quiz

  • Ensure questions are clear and easy to understand with a single answer (no trick question)

  • Ensure learners know how to answer the question (put items in order, select one, select many)

  • Provide feedback for both correct and incorrect answers (does the learner retry until they get it correct?)

Instructional and Interactive Content

  • Connect content to syllabus and learning objectives

  • Ensure learner knows what to expect and how to progress (does the quiz count towards their grade? does the instructor keep track of completion?

  • Take existing code logic into consideration

Tone and Word Choice

  • Use a confident, friendly, and respectful tone

  • Write clearly, concisely, and specifically (aim for a grade 8 reading level, shorter words and sentences to avoid confusion)

  • Use terminology consistently and correctly (use Google or ask content expert for clarification)

  • Do not use the development terms such as VTS, Unity or CORE (learners don’t or care about the tools)

  • Use course name when necessary (Welcome to machine shop safety orientation), but make generic when possible (allows reuse)

  • Avoid gender-specific and ableist language

  • Avoid culture generalizations and stereotypes
