VARLab : Amending Commits

Created by Jordan Robinson, last modified on Jul 05, 2022

Sometimes you need to make a small change to your branch but you don’t think it warrants a whole new commit. In this case we can amend your previous commit in order to rewrite history as if your change was there the whole time.

In Sourcetree under the File Status tab we can view our changes, stage our changes like usual, but before we go to commit we need to select Amend Latest Commit in the bottom-right.

Once you’ve selected this option you’ll notice the previous commit message shows up in the message box.

After clicking Commit you’ll find that you have a commit to push and pull.

This is because your local branch is now out of sync with Bitbucket and we need to tell it that our version is the true version with a Force Push. If force push is not enabled, you will need to enable it in Tools > Options > Git > Enable force push first.

After force pushing, your previous commit will be amended as if the changes you just added were always there to begin with.
