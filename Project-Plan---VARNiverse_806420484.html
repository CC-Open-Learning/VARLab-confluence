VARLab : Project Plan - VARNiverse

Created by Nick Harrison, last modified by Brodin Collins-Robb on Sept 25, 2023

Features From Research

  • lost in the woods as the hub world

  • character controller

  • hub world environment

  • intro to scene with skipping capability

  • teleportation

  • VR capability

Unity Asset Packages Used

All packages in this project will be free packages

https://assetstore.unity.com/packages/3d/environments/lowpoly-environment-nature-free-medieval-fantasy-series-187052#content - Main hub world

https://assetstore.unity.com/packages/3d/environments/fantasy/medieval-town-exteriors-27026 - Main hub world

https://assetstore.unity.com/packages/3d/environments/sci-fi/sci-fi-styled-modular-pack-82913 - potentially main hub world? or inspiration (easter egg in the main hub where more teleportation stones can be added)

https://assetstore.unity.com/packages/3d/environments/sci-fi/free-demo-of-low-poly-space-alien-worlds-3d-asset-pack-258683#description - would mainly be used as a world to walk around in to spark ideas

https://assetstore.unity.com/packages/3d/props/tdg-italian-ice-cart-257881 - to add detail to the main hub

Character Controller

  • arrow keys or WASD to move around the character

  • First Person Perspective

  • use mouse click to interact

  • have collision detection with both mouse and capsule

Hub -World Environment

  • The hub world environment should be fairly large so that many world stones can be added

  • This hub should include buildings and trees to have an interesting environment with lighting effects and skyboxes to create a very interesting environment for members to explore

  • some where in the world have a space base from the space package in a hill or cliff side

  • have a small town using the medevil package

  • colliders will be needed on all OBJ in this Hub world

Intro Scene

  • guides the user with instructions for navigating the open sandbox and basic instructions on entering and editing the sandbox

  • Skippable in case the user has seen it before

  • shows the player model and overall sandbox

Teleportation

  • Teleportation will be triggered when a user walks up to a stone in the main hub, and they will be able to view a popped-up dialog box in the UI above each stone to read about the mini-game they would be teleporting to

    • This will help make the process of creating a mini-game easier for new onboarded members to get started with their mini-game as they will be able to edit UI text to represent their game rather than need to modify an asset for the game.

  • the teleport will trigger when the player clicks the mouse

  • the pop up of name of the mini game will trigger when a ray cast from the players mouse is touching the stone

VR Capability

  • We could set frameworks to allow VR minigames to be implemented

  • add VR Movement to the player controller to allow the players to explore the hub world in VR as well as have a base player control to use for making a VR on boarding or jam project

Example Minigame

  • To help new onboarded members start creating their own mini-games, we will have some pre-created mini-games to look off of in terms of examples

  • Brodin has a mini-golf mini-game from his onboarding that could be used as an example minigame to help fast-track the process of development

Stretch Goals (Mini Game Ideas)

  • use other free environment packs to build mini-games to help spark ideas of Jammers/On-boarders

  • input playable doom?

  • have one mini game that uses node canvas(if licensee allows it) and demonstrates how VAR lab does DLX question and answer flow. this can be made to have both VR and 2D screen.
