VARLab : Testing For a Build Server

Created by User aa77f on Jun 04, 2024

As we Introduce DLX to the Pipelines, our Unity experiences will be running tests and building on a remote server to automatically run tests. The result of this is that we need to account for tests running in an environment with no graphics/display. A way to work around the limitations of the build server is to introduce the idea of Test Categories.

What is a Test Category?

This is essentially tagging individual tests with a string, so when we run the tests on the server we run only the subset of tests that can run without issue! They are simple to add as shown below.

Just add [Category("BuildServer")] under [UnityTest] for each test and the tests are now categorized!

When to add the category?

For now every PlayMode test should have the category added to it. EditMode tests can be left unchanged as they run using the Nunit framework.

When to remove the category?

This is important, we remove the category to tell the build server to exclude tests from running on it!
