VARLab : Functional Testing (DRAFT)

Created by Sana Javeed, last modified on Oct 10, 2024

This testing is conducted to determine whether the application is working according to the specifications.

Some examples of the common functionalities are stated below.

  • Multiplayer Test Criteria
  • Settings UI
  • Login page

Multiplayer Test Criteria

Pre-condition: DLX supports multi-player mode

  • If there is one game session?

    • Verify that multiple users can join the same game session.

  • If there are no game sessions but allows to form teams

    • Verify if the user can form teams and join existing teams

  • If one user can invite other participants to a game session?

    • Verify that multiple users can invite participants to a game session

    • Test if the process is simple and a success

    • A success is when a user has sent an invite and the participants who have been invited can successfully join

  • Verify that the users can communicate with each other through in-game chat, if any

    • Test if users can send and receive messages

    • Test if the users are notified when a message is received

    • Test if the messaging supports speech to text functionality along with text support

  • Verify users can view other users' game stats and leaderboards, if any

Settings UI

  • Verify the access to settings such as sound, volume and sensitivity

  • Verify the performance and response on the desired impact after making changes to the settings

  • Verify the save and load of user preferences

Login page

Pre-condition: Users must have an existing google or microsoft account

  • Verify that the page supports social logins like google and /or microsoft

  • Verify that the user can log in successfully with thier existing accounts
