1: TexTools Description


TexTools for Blender is a great free tool for working on UV’s and running various texture bake procedures - originally developed for 3DSMax but is now available for Blender. The tool has several useful features, including:

  • Multiple out-of-the-box Texture Baking modes

  • UV Layout Tools (Align, Rectify, Sort, Randomize, etc)

  • Texel Density Tools

  • Smart UV Selection operators

  • Color ID tools

  • (some) UV-related Mesh Creation utilities


2: Features Breakdown

Size and Padding

  • Set the texture size (in pixels) for baking and checker maps

    • Size Dropdown: Menu with several common Image sizes

      • 32 → 64 → 128 → 256 → 512 → 1024 → 2048 → 4096 → 8192

    • Padding: Sets the amount of padding generated at the edge of each UV island when baking

UV Selection, Texture Reload, Checker Map

  • UV Selection: Change the UV channel currently being used

  • Reload Texture: Reload all textures

  • Checker Map: Add a checker map to the selected model and UV view

UV Layout Tools

  • Crop

    • Crop UV area to selected UV Faces

  • Fill

    • Fill UV selection to UV Canvas

  • Align Edge

    • Align the island by selected edge

  • Align World

    • Align selected UV islands or faces to world/gravity directions

  • Directional Alignment Tools

    • Align vertices, edges or shells

  • UV Rotation, Mirroring, and Sorting

    • Rotation: Rotate the selected UV island 90 degrees left or right

    • Mirroring: Mirror selected faces either Horizontally or Vertically

    • Sorting: Rotate UV Islands to minimal bounds and sorts them Horizontally or Vertically

  • UV Centralization and UV Randomization

    • Centralize: Move selected islands the closest possible to the 0-1 UV area without changes in the textured object

    • Randomize: Randomize UV Islands/Faces Position and/or Rotation

  • UV Straightening

    • Straighten selected edges chain and relax rest of the UV Island

  • UV Rectify

    • Align selected faces or verts to rectangular distribution

  • UV Unwrap

    • Unwrap selected UV’s

  • UV Relax

    • Relax selected UV verts

  • UV Edge Peel

    • Unwrap pipe along selected edges

  • UV Iron Faces

    • Unwrap selected faces into a single UV island

Baking Tools

AA

Anti Aliasing on Baked Textures

  • Options: 2x and 4x

  • Slows down render time but the lines of outputted textures are extremely clean in appearance

CS

Color Space used for Texture Bakes

  • sRGB: Standard RGB output color space for the baked texture

  • Linear: Linear (or non-color) output color space for the baked texture

Alpha

  • Bakes the Alpha transparency pass of a material

  • Color Space: Linear (Non Color)

Anisotropic

  • Bakes the Anisotropic pass of a material

  • Color Space: Linear (Non Color)

Anisotropic Rotation

  • Bakes the Anisotropic pass of a material with Rotation

  • Color Space: Linear (Non Color)

AO

  • Bakes an Ambient Occlusion map with the Cycles Renderer and with normalized enabled

  • Samples: The amount of global illumination samples

  • Color Space: Linear (Non Color)

If working with procedurally generated textures you are able to use this mode to bake the ao map of your texture with minimal setup - just make a material on your model, set the texture to it, then bake

Base Color

  • Bakes the Base Color pass of a material

  • Color Space: sRGB (Color)

If working with procedurally generated textures you are able to use this mode to bake the color map of your texture with minimal setup - just make a material on your model, set the texture to it, then bake

Bevel Mask

  • Bakes a mask using the bevel shader

  • Edges are solved via shaders and is geometry independent

  • Bevel Samples: Number of Samples (1-16) rendered, the Higher the Better (but slower)

  • Radius: The radius of the bevel in scene units

  • Color Space: Linear (Non Color)

Cavity

  • Assigns dirty vertex colors (convexity) to the source mesh and combines it with a “pointiness” cycles render node

  • This node depends on the topology of the model

  • Color Space: Linear (Non Color)

Clearcoat

  • Bakes the Clearcoat pass of a material

  • Color Space: Linear (Non Color)

Clearcoat Rough

  • Bakes the Roughness of the Clearcoat pass of a material

  • Color Space: Linear (Non Color)

Combined

  • Bakes all materials, textures and lighting (except specular) into one texture map

  • Passes that contribute to the combined pass can be toggled individually to form the final map

  • Lighting:

    • Direct

    • Indirect

  • Contributions:

    • Diffuse

    • Glossy

    • Transmission

    • Emission

  • Samples: Number of Samples (1-4000) rendered, the Higher the Better (but slower)

  • Color Space: sRGB

Curvature

  • Renders a tangent normal map and converts it to a curvature map using compositing nodes

  • Curvature: The thickness of pixels of the curvature offset effect (1-64) the Higher the Better (but slower)

  • Note: Requires a High Poly object and a Low Poly object

  • Color Space: Linear (Non Color)

Diffuse

  • Renders the diffuse colors of all Cycles materials of the object

  • Requires the Cycles Renderer be set as the Active Renderer (not Eevee)

  • Color ID’s should be rendered in this mode

  • Color Space: sRGB

Dust

  • Dust is similar to the cavity map but adds additional Z top normals to the mix

  • Use this mask for dust and dirt masks in your texturing process

  • Color Space: Linear (Non Color)

If working with procedurally generated textures you are able to use this mode to bake the dust map of your texture with minimal setup - just make a material on your model, set the texture to it, then bake.

Note that you will need to combine maps in Image Editing

Emission

  • Bakes Emission (or Glow) color of a material

  • Color Space: Linear (Non Color)

Emission Power

  • Bakes the power of the Emission color of a material

  • Color Space: Linear (Non Color)

Be sure that your Emission Power is set to 1.0 when baking → When applied to materials you should be able to increase and decrease the strength through the particular editors settings

Environment

  • Bakes the environment (world surface shader defined for the scene) onto the selected object(s) as seen by ray casts from the world origin

  • Color Space: sRGB

Glossiness

  • Bakes the glossiness pass of a material

  • Color Space: Linear (Non Color)

Element ID

  • Bakes a colored texture where each mesh element has a unique color

  • Use this map to isolate different parts of your model in the texture

  • Color Space: sRGB

Material ID

  • Assigns a unique color to each sub material or material slot

  • Color Space: sRGB

Metallic

  • Bakes the Metallic pass of a material

  • Color Space: Linear (Non Color)

If working with procedurally generated textures you are able to use this mode to bake the metal map of your texture with minimal setup - just make a material on your model, set the texture to it, then bake.

Object Normal

  • Bakes a normal map in object space

  • Use the green channel for vertical gradients

  • Color Space: Linear (Non Color)

Object Normal Bevel

  • Bakes a normal map in object space with beveled edges

  • Bevel Samples: Number of Samples (1-16) rendered, the Higher the Better (but slower)

  • Radius: The radius of the bevel in scene units

  • Color Space: Linear (Non Color)

Tangent Normal

  • Bakes a normal map in tangent space

  • Can change the Y direction of normal map in the Addon Preferences

  • Color Space: Linear (Non Color)

  • Note: Requires a High Poly object and a Low Poly object

Tangent Bevel

  • Bakes a normal map in tangent space with beveled edges

  • Bevel Samples: Number of Samples (1-16) rendered, the Higher the Better (but slower)

  • Radius: The radius of the bevel in scene units

  • Color Space: Linear (Non Color)

Paint Base

  • Sometimes referred to as Blizzard Diffuse Map or DOTA Diffuse Map

  • This bake combines a dirt map with vertical gradient + top facing normals with a highlight

  • This bake is a good starting point for character texture maps

  • Color Space: sRGB

Position

  • Bakes a map with unique colors with the bounds of the object

  • Color Space: sRGB

Roughness

  • Bakes the roughness pass of a material

  • Color Space: Linear (Non Color)

If working with procedurally generated textures you are able to use this mode to bake the roughness map of your texture with minimal setup - just make a material on your model, set the texture to it, then bake.

Face Selection

  • Bakes a black and white mask of the face selection on the object

  • Can be useful to generate quick masks for working with textures

  • Color Space: Linear (Non Color)

Shadow

  • Bakes shadows and lighting

  • Samples: Number of Samples (1-4000) rendered, the Higher the Better (but slower)

  • Color Space: Linear (Non Color)

Sheen

  • Bakes the Sheen pass of a material

  • Color Space: Linear (Non Color)

Sheen Tint

  • Bakes the Tint of the Sheen pass of a material

  • Color Space: Linear (Non Color)

Specular

  • Bakes the Specular pass of a material

  • Color Space: Linear (Non Color)

Specular Tint

  • Bakes the Tint of the Specular pass of a material

  • Color Space: Linear (Non Color)

SSS Color

  • Bakes the Sub Surface Scattering Color pass of a material

  • Color Space: sRGB

SSS Strength

  • Bakes the Sub Surface Scattering Strength pass of a material

  • Color Space: Linear (Non Color)

Thickness

  • Bakes the Thickness pass of a material

  • Samples: Number of Samples (1-4000) rendered, the Higher the Better (but slower)

  • Distance: AO distance

  • Contrast: AO contrast

  • Only Local: Only detect occlusion from the object itself, and not others

  • Color Space: Linear (Non Color)

Transmission

  • Bakes the transmission pass of a material

  • Color Space: Linear (Non Color)

Transmission Roughness

  • Bakes the transmitted roughness pass of a material

  • Color Space: Linear (Non Color)

UV

  • Mapped UV coordinates, used to represent where on a mesh a texture gets mapped to

  • This is represented through the red and green channels of the image

  • Blue channel is encoded with a constant value of 1 but does not hold any information

  • Color Space: sRGB

Wireframe

  • Bakes a black and white wireframe mask

  • Thickness: The thickness of the wireframe in pixels

  • Color Space: Linear (Non Color)

3: VAR Lab PBR Baking Processes

  • Ensure that TexTools is installed and enabled in Blender first

  • When ready to bake, open up the UV map editor and expand the TexTools tab on the right of the window

  • Note that the following processes are based upon the assumption that you have already completed UV mapping

Step 1: Prep TexTools Panel Settings for Baking:

  1. At the top of the TexTools panel set the Size to be the target size for the image

    1. Usually either 512 or 1024

  2. Set the padding value to be 2 → 4 (4 is default)

  3. Set up your Baking Mesh so that it is properly exploded for baking purposes (see Asset Developers Ian or Topher for instructions on how to do this if you are unsure).

    1. You may also wish to component-ize your mesh (ie: split up the geometry into multiple meshes, especially if several sections are touching eachother - this often is the cause of distortions in bakes)

  4. Ensure the following:

    1. Scales and Rotation have been applied

    2. Normals are all facing the correct direction

    3. The Origin Point(s) have been properly set

    4. Smooth Shading and Auto Smooth are properly set up

Step 2: Prep Mesh for Baking

1: In your Blender Hierarchy Panel (Top Right), create a new collection labelled “Bake”

2: In the new “Bake” collection create an additional 2 new collections, 1 labelled AO and the other labelled Normal

3: Select the original UV-mapped Mesh you wish to bake and Duplicate it twice, placing 1 Duplicate in the Bake-AO collection and 1 Duplicate in the Bake-Normal collection. Now Deactivate the collection containing the Mesh original

4: Select the Meshes in the Bake-AO collection and Bake-Normal collection and increase the Mesh scale by a factor of 10 (originally large meshes, like a car) → 50 (originally small meshes, like a pencil).

  • Scale Function: S Key → [Type in #] → Enter

Apply the scale to both meshes once done (so that the bakes will run correctly)

  • Apply Scale: Object Tab → Apply → Scale

The reason for scaling meshes up before baking is that it leads to far cleaner bakes in the tool (due to an increase in information being captured by the increased size). To see the difference, perform the following baking functions without performing the Scale Function and then try it again with the Scale Function - you will quickly notice a marked difference between the two.

Step 3: AO Map Baking:

  1. Expand the Baking Panel and click on the picture of the Purple Cylinder to bring up the Bake preset menu - Locate and click on the White Cylinder with the caption “AO” to activate the tool.

This tool automatically bakes your Ambient Occlusion Maps.

  1. Click on the dropdown next to AA (Anti-Alias) and set the value to 4x

  2. Click on the dropdown next to CS (Color Space) and set to Linear (instead of the default sRGB)

    1. Linear = Non Color

  3. Set your Samples to a desired value

    1. Generally 128 Samples works quite well

  4. Now select your Mesh and enter Edit Mode and Separate the Mesh by Loose Parts so as to componentize the Mesh (anything not connected geometrically will be made an individual object)

    1. A Key (Select All) → P Key (Separate) → By Loose Parts

  5. Once Componentized, further separate the Meshes based upon the following 3 classifications:

    1. Exterior Surfaces

    2. Interior Surfaces

    3. Inbetween Surfaces

Using a Pouch for a Bag as an example, the Exterior and Interior faces would be self explanatory while the Inbetween Faces would be areas like the Ring of faces separating the Exterior and Interior Faces (aka the border)

Note: If no Interior and/or Inbetween Surfaces exist then this step can be skipped

  1. Once all Faces have been separated per the above workflow, go back into Object Mode and then manually explode all of the objects by spacing them all out so no object is able to cast shadow on another (unless otherwise combined together) - having 5m x the Scaled Factor that you used in Step 2 should be sufficient

    1. TexTools explode function would still function correctly when in the AO bake mode, however the Explode function only moves meshes by a set amount but still keeps the objects relatively close together - due to the spacing, when running the bake these other objects will cause unnecessary shadows on nearby objects. However, due to the relatively small number of mesh surfaces for this bake procedure, it makes manually exploding the mesh(es) a very short procedure and should only take moments)

  1. Ensure that the Single Texture option is set, as this will ensure that all components are baked onto a single texture (instead of textures being generated for multiple different parts)

  2. When all settings have been properly set, click on the Bake 1x button and wait for the baking procedure to complete

  3. If there are any errors with the bake, make adjustments to the UV’s and/or the Mesh where needed and then re run the bake (and repeat this process until a suitably clean bake is output).

    1. Be sure to save your bake as [TextureSetName]_Bake_AO through the Image Editor

    2. If you continue to have errors and distortions in your bake contact Asset Developers Ian or Topher for assistance

Step 4: Normal Map Baking:

  1. Expand the Baking Panel and click on the picture of the Purple Cylinder to bring up the Bake preset menu - Locate and click on the Purple Cylinder with the caption “Tangent Bevel” to activate the tool.

    This tool bakes a Normal Map by baking Bevels onto the UV map around the edges of the UV islands and completely removes the need to go through a high poly → low poly baking process entirely

  2. Click on the dropdown next to AA and set the value to 4x

  3. Click on the dropdown next to CS and set to Linear (Linear should be enabled by default)

  4. Set the Bevel Samples to 16

  5. Set the Radius value to 0.01

  6. Now select your Mesh and enter Edit Mode and Separate the Mesh by Loose Parts so as to componentize the Mesh

  1. Once you have componentized your Mesh, go through each Object and further separate those meshes out by their Surface Borders (edges of a face surface group where a “hard” edge would be placed, this is where you want your Bevels to be baked per the UV’s)

    1. Cube Example: All 6 faces of a cube would be separated out into 6 individual faces

  2. Once Separated out, Select all Objects in the Bake-Normal Collection, and Set their Origins to Geometry (Center of the Individual Mesh)

  1. In the TexTools Panel, locate the “Bake Sets” panel underneath the Bake Mode panel and set “Force” from “Normal” to “Multi” and then click on the “Explode” button → This will automatically move all selected Meshes out along the X, Y, and Z axis’s so that no Mesh element is overlapping another. This is accomplished by the Tool creating Keyframes at Keyframe 0 (Original location of Mesh before you clicked Explode) and Keyframe 50 (position that the Tool exploded the Mesh Element to)

    1. If you slide the Animation Timeline between the 2 keyframes you can see animation play out in real time

  1. Now turn on Face Orientation Mode in the Overlays Panel, and then Select all Objects in the Bake-Normal Collection

  1. Go into Edit Mode, Select all Faces, Extrude with the E Key, and then right click to snap these faces back to their original positions → Do not deselect these faces as they will be needed in a moment

  1. With the Faces still selected, Scale these faces out on their Normals with the keys Alt+S and then using your mouse adjust the placement of the Selected Faces so that the Mesh turns from Blue (Normals facing outwards) to Red (Normals Reversed) and so that a small overhang of faces is created (appearing like a Bevel would) → Hit enter when Satisfied

    1. IMPORTANT: Before doing anything else, when performing the above detailed extrusion function you will notice that once you hit the Enter key the faces you had selected will still be selected → You now want to delete these Selected Faces → Once the selected faces are deleted, select all Faces and then recalculate their Normals so that they point outwards correctly (Shift N)

  1. Combine the Meshes together and then Ensure that the Single Texture option is checked on.

  2. When all setting have been properly set, click on the Bake 1x button and wait for the baking procedure to complete

  3. If there are any errors with the bake, make adjustments to the UV’s and/or the Mesh where needed and then re run the bake (and repeat this process until a suitably clean bake is output).

    1. Be sure to save your bake as [MeshName]_Normal through the Image Editor

    2. If you continue to have errors and distortions in your bake contact Art Leads Ian or Topher for assistance

If you find that your bevels are too subtle, try increasing the Radius value in the Bevel Bake Panel from the default 0.05 value to 0.10 or more and then re run the bake

Step 5: Confirm Bake Appearance on Original Mesh with Material

Add a Material to the Original Mesh and recreate the following Node Setup to Confirm all bakes appear correctly on the mesh

Notes for Troubleshooting

  • If baking on surface details (like screws, rivets, etc) and the bake results in a Normal Map section appearing inverted, all you need to do to fix this is to move the related mesh section(s) off of the surface by a value of 0.001 (wherever the “Up” Normal direction is), rotate the section by 180' on the Z axis (essentially inverting the mesh), flip the normals, and then you should be able to re-run the bake!

  • For Normal Map Baking, sometimes Mesh Elements will also not bake onto the Map if they are overlaid on another surface (a button on a control panel, for example). To solve this:

    • 1: First save your initial bake that you ran with a proper label

    • 2: Now go into your bake mesh and separate out the surfaces that did not bake (the button on the control panel in the example just mentioned) so that they are a separate mesh object from the rest of the control panel mesh.

    • 3: Now hide everything except the surfaces that didnt bake

    • 4: With the surfaces that didnt bake selected, re run the Normal Map bake and you should see the bake run correctly

    • 5: Save the new bake image as [MeshName]_Bake_Normal_Add (or something to that effect)

    • 6: Open your image editor, open the 2 Bake maps, and then on the Add map use the magic wand to remove everything except for the elements that got baked from the image

    • 7: Copy the edited Add map and paste it as a new layer on the base original map you saved (not the Add map) and then save the image. This will layer the normal map elements cleanly!
