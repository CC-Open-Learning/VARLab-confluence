Purpose

The primary goal of this research is to explore and propose suggestions for optimizing the use of Jira plugins for test case management. By evaluating various Jira plugins, the aim is to determine which plugins will provide the most efficient, organized, and streamlined process for managing test cases within VARLARLab's testing environment.

Current Test Case Management Practices at VARLab

At present, the test case preparation process in Varlab is fully manual. Once the basic test cases are documented and shared via Confluence, the team lead creates corresponding Jira tickets as user stories. These tickets are not assigned to any team members, instead everyone who will be executing the test cases post their findings as comments under each Jira ticket.

The status of each task is determined based on these comments. If any issues are identified during testing, separate Jira issues are created and linked to the respective test cases.

Currently, test cases are not reusable, which adds to the time and effort involved in preparing and executing tests for similar functionalities in future development cycles.
