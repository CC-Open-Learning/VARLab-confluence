VARLab : Blender 4.2 Addon "PBR Painter 3.0" General Guide

Created by Ian Kennaley, last modified on Jan 08, 2025

This guide is based on using Blender 4.2 and version 3.0 of the PBR Painter Addon. This will be a basic setup and use guide and is not necessarily meant to be exhaustive for absolutely all use cases.

As Blender and the Addon both develop, updates to this article will naturally need to be made.

Step 1: Addon Installation

  • Ensure that you have downloaded the addon and have gone through the installation procedure

    • Contact IT regarding being provided a license to use the addon

A) Go to Edit → Preferences to access the Addon and Extension interface

B) Click on Addons to open the Addon list

C) Click on the Down Arrow at the top right and click “Install from Disk”

D) Locate the Zip file containing the PBR Painter 3.0 addon and click confirm

E) In the PBR Painter Addon dropdown set a Folder location to store any Material Assets you might want to use or store (note: works through Blenders Asset Browser), and then click on the Load PBR Painter Workspace button to generate the addon’s Workspace Tab in the Blender main window

F) Confirm that the PBR Painter Workspace was created. Once confirmed, you should now save the Blender file as the Default Startup File to ensure that the Workspace is present in all of your future projects

Step 2: Create your Model and ensure that it is UV Unwrapped, and generate any AO and Normal Map Bakes (HP → LP Baking, TexTools baking, etc)

  • Ensure that you have properly UV Unwrapped your model before beginning work in the addon to ensure you're able to cleanly texture your model

  • If you are using a AO/Normal Map bake approach to your model, ensure you have completed these first before proceeding.

  • For information on HP->LP AO/Normal baking please review the 3D Asset Production Pipeline - Version: Beta - VARLab - Confluence (atlassian.net) and for an alternate approach to creating these maps please review the Blender - TexTools Addon Guide - VARLab - Confluence (atlassian.net). These will provide some guidance on how to use each method (although not exhaustive)

Step 3: Duplicate your UV Unwrapped Mesh, Clear any materials on the mesh, and then create a new Material on the Mesh from the PBR Painter Interface

  • It is good practice to utilize Blenders Collections System to keep things organized

  • Create a new Collection and call it something like “PBR Texturing” or “Texturing” (something descriptive)

  • Duplicate your UV Unwrapped mesh and rename it to have a “_Texturing” suffix and move the duplicate into the Texturing collection

  • Open the PBR Painter addon tab and then remove any Materials that might be present

    • Either by unlinking the Material, or deleting the Material slot entirely

  • Create a new Material with the “New” button in the Painters interface

    • PBR Painter uses a specific custom node setup to make everything work - if you use any other material and node setups then the addon won’t function correctly

  • Name the PBR Painter Material to be the same as the Asset but do not include the “poly_” prefix

1: Create new collection and select mesh

2: Duplicate mesh and put duplicate into Texturing collection

3: Open PBR Painter tab and remove any materials on Mesh

Alternatively delete the slot

4: Create new PBR Painter material by clicking the New button

5: Rename the new Material to be the same as the mesh, minus poly_, and use the _Texturing suffix

Step 4: Add any associated AO/Normal map bakes as a new Layer and ensure everything is configured correctly

Note that this step is optional if you are not utilizing AO/Normal map bakes from HP → LP Bake workflows, TexTools, etc

  • To utilize pre-baked AO and Normal maps from HP → LP baking, TexTools, etc, first add a new Layer by clicking on the Add Layer button

  • Rename the new Layer to “Layer_AONormal” or something similar

  • Enable the Material Auto Refresh to ensure that the Material is updated as you add and configure layers

    • Note: You can leave this off and manually update the material if you so desire (although this can get tedious as you configure things, and tweaks can take a bit longer as well)

  • Click on “Multi Import” and navigate to where the baked AO and Normal Maps of the mesh are stored and import said files

  • The bake maps should be auto assigned to the AO and Normal channels of the Layer. Some additional configuration should be made for the channels:

    • Color Channel:

      • In addition to the AO map being applied to the Color Channels “AO” Color Correction option, you should also set the AO map to the Color Channels base Color layer and set the Channels Blend mode from “Mix” to “Multiply” and the Color Space from RGB to Non-Color.

      • This will give the benefit of baking down the AO map details onto your color maps so as to better make things pop.

    • Normal Channel:

      • Ensure that the “Combine Normals” option is enabled so as to mix the baked Normal Map you imported with the Normal Maps of the tile texture(s) you will use to set up your other layers

As you add additional layers, it is recommended that you keep the “Layer_AONormal” layer at the following positions for their associated results:

Top:
AO map is combined and with the color channels of the other Layers and then baked down (in addition to the generated AO, MRO, and ORM maps during the final texture export covered later in this guide)

Bottom:
AO Map is not combined with the Color map (but is still used for the AO, MRO, and ORM map generations)

1: Add a new layer

2: Rename the layer to “Layer_AONormal”

3: Enable Automatic Refresh of the material

4: Use Multi Import to import the AO and Normal map bakes

5: Ensure you have your AO map applied to the Color channels Base and set the Blend mode to Multiply

6: Ensure that you have enabled Combine Normals on the Normal Channel of the layer to enable combination with the Normal maps of other layers you create

Step 6: Enable the Layer ID Map option for the new Layer and use the Color Palette in the Image Editor panel to pick Colors to set the Color ID for surfaces to have Layers applied to

Note:
While technically this step is optional for the Kidney Basin asset (as its all 1 continuous material), this Step should be performed at this stage of the Texturing process for assets that have multiple layers of different surface types being used

This is because PBR Painter allows you to use Color ID’s to define what Layers are being applied to the various surfaces of your model (similar to how Mesh Masking works for Substance Painter materials)

  • With the Layer you are setting up selected, Enable the Layer ID Map option and expand the heading

    • The model should now only display the Layer for your AO/Normal map (Step 4 of this guide) as that layer is not set to use Layer IDs, and therefore it gets combined with all other Layers when everything is Baked down

  • Keep the mode on Vertex Color and click the “Create New” button

    • Note the “Paint Color” and “ID Color 1” Color selection panels - these are where you will be setting the colors for the Layer ID

  • In the 2D Painting Panel (bottom Left of PBR Painters Workspace default layout), expand the Tool panel and navigate to Tool tab → Brush Settings → Color Picker → Color Palette, and expand the field

    • Set the Color Palette selection to the built-in Blender Default palette

    • In the Color selection field, pick one of the colors, locate the Hex Code field, and then copy the code in the field

  • In the Paint Color field of the Layer ID Map, paste the Hex Code you just copied and hit Enter

  • In the ID Color 1 field of the Layer ID Map, paste the same Hex Code you copied and hit Enter

  • Now go Into Edit Mode on the model (default: Tab Button)

  • With Face Select enabled, Select the Faces of your mesh you want to have the Layer be applied to

  • In the Paint Color field, use the “Fill Selected” Button to to apply the Color to those Surfaces

    • Visually, you should see your model update the Material instantly if you have Auto Refresh enabled. If not, Click the Refresh Material button and you should see the model surface update and display the textures

    • If you need to visually see the Color ID’s displayed on your model:

      • Change to Solid Shaded View → Viewport Shading Dropdown → Color settings → Set to Attribute

      • The ID Map should be displayed now on the model

Step 7: Adjust the Layers Image Mapping by adjusting the Layers Projection Settings (UV based or Box Projected) and the associated Position/Rotation/Scale Mapping values

  • PBR Painter allows you to be able to adjust the way that the Textures are being mapped to your model, either by:

    • A) Based on the the UV’s of the model, or

    • B) By projecting the Texture onto the Model as a cube, and translating that information to the UV’s of the model

  • Start by navigating to the PBR Painter master panel

  • With the Layer you are adjusting selected, Select one of the Texture channels of the Layer (Channel selection doesn’t really matter, just needs to be one of the channels of the Layer)

  • Expand the Mapping Heading to access the settings

  • It is generally recommended to change the Mapping mode from UV → Box (PBR Painter)

    • This projects the Textures onto the model as a cube and then transfer that information to the UVs of the model to create the Texture maps during baking

  • The seams from the Texture projection become very apparent as the Seam Blend is set to 0 by default

    • The seams are the noticeable hard edges of the texture projection

    • To adjust this, a good start point is a value of 0.30 → From there, make further adjustments until the seams of the texture mapping disappears on the model sufficiently

  • Now adjust the Scale Values of the Mapping to Tile the Layer’s Textures across the model

    • Try to avoid tiling the Textures too heavily as the repetition of the seams becomes more noticeable as you tile it more heavily

  • Adjust the Rotation values as needed to rotate the projection of the Textures onto the Mesh

  • Adjust the Location values as needed to adjust the positioning of the Textures being applied to the UVs of the model

  • The Texture mapping process applies the same way to the UV Mode of the Image Mapping, just based on the actual UVs of the 2D UV map and not projecting based on the 3D view

    • It is often a good idea to switch between the 2 modes when you find your Textures are not tiling well in one way or another

1: Navigate to the main PBR Painter Panel (far right)

2: Expand Mapping panel

3: Locate the Mapping heading

4: Change the Mapping mode from UV [default] to Box (PBR Painter) [recommended]

5: Note how the Textures are now displayed on the Model

6: Adjust the Seam Blend value from the Default 0.00 and set it to 0.30 (and adjust further as needed)

7: Adjust the Scale of the Texture Mapping on the model to give more detail and definition of the texture

Note the way that the textures are scaling on the Mesh surface as you adjust the values

8: Adjust the Rotation values if needed to rotate how the Textures are mapped to the model

In this case, we can rotate the Z axis projection to change the lateral tiling from the Y axis to the X axis (without needing to adjust the UV’s and rebake any AO or Normal maps and then re setup the PBR Painter layers)

Step 8: Set up a new Layer for Painting on Decals, Set the Brush Color to White, Enable the Brush Texture option and set the Decal as the Image and adjust the settings, and then Position the Decal and paint it on

Controls for Manipulating Decal Images
PBR Painter allows you to manipulate your imported Decal Images within both the 3D Viewport and within the 2D Image Editor

Left MB Click: Paint

Right MB Click + Hold: Move Decal Image around

Shift + Right MB Click + Hold: Scale Decal Image

Ctrl + Right MB Click + Hold: Rotate Decal Image

For the purposes of this guide, a decal for a Kidney Bean was made and has been provided above for the purposes of learning how to use the Decal System. You may use any other decal image that suits your preference if so desired!

  • To paint a Decal (Stickers, Labels, etc) onto a 3D model, you first need to create a new layer

    • Label the Layer as something akin to “Layer_Decal”, something descriptive

    • Set the Fill type of the new Layer to “Painted” and enable the 32 Bit Float option

  • In the main PBR Painter window (far right) scroll down to the Image Header and expand it, and then click on the “Paint” button

  • In the Left Side “PBR Painter Brush Settings” panel, under Brush Color click on the “White” Button

    • The White and Black brush colors are essentially “Alpha” (Black) and “Non-Alpha” (White) and tells PBR Painter when to apply a Decal stroke on the layer

    • In short: When painting decals on, be sure to use the White Brush color

  • Just below the Brush Color Settings enable the setting “Brush Texture” and expand the heading

  • Click the button labelled “New+” and a new Texture entry will be created

  • Change the Mapping setting from “Tiled” to “Stencil” to enable the painting on of the decals (like using Stencils for Spray Painting)

  • Click on the “Open” button and navigate to where you have the Decal Image stored and import it

  • You should now be ready to paint your Decal Image on. The Decal Image will show up in both the 3D view and the 2D view as a Transparent Overlay

  • Using the controls listed at the top of this Step, you can Move, Scale and Rotate the image in either Viewport (3D or 2D) to position the Decal on your model and then Left Clicking to apply the Decal with strokes

    • If you find you need to reset the Rotation of your Decal, expand the “Mapping” heading and adjust the Rotation Value as needed

    • You can use the Radius Slider to adjust how large your Brush is, and Strength affects how hard the Decal applies

  • You can use the controls in the Image Sampling heading to make adjustments that affect how cleanly the Decal paints on

2: Name the Layer descriptively, set the layer from “Filled” to “Painted”, and ensure 32 Bit Float is enabled

Step 9: Change to the Bake Tab, adjust the Baking settings as needed, and set the Material Export Settings

  • When ready to combine your layers and bake out your final Textures, switch to the Baking Tab of the main PBR Painter panel

  • Expand the General Bake Settings heading and untick “Include Resolution in Filename” so that when you bake your textures out you do not have extra text added to your labels

    • You may try using the “Save Textures Externally” option, however you lose the ability to control what Color Space your textures are exported as (ie: They are all exported as sRGB, where as you only want your color to be sRGB, while the rest of your maps you should be set to Non-Colour)

  • Now expand the Material Exporting Tab

    • Set your Resolution to 1K

    • Set your Margin (bake padding) to 2px

    • Set your Bake Samples to 64

  • Now change the Channel Selection option from “Automatic” to “Manual” and enable the “Channel Packing” option

  • Use the Plus Sign button to add 8 additional Channel-packed Images and rename all of the entries as follows:

    • _colour

    • _ao

    • _normal

    • _metal

    • _rough

    • _smooth

    • _MS

    • _MRO

    • _ORM

  • Now go through each Channel Packed Image entry in the list and set them to as follows from the dropdowns:

    • _colour

      • 32-bit float: Enabled

      • Color: Base Color

    • _ao

      • 32-bit float: Enabled

      • Color: Ambient Occlusion

    • _normal

      • 32-bit float: Enabled

      • Color: Normal

    • _metal

      • 32-bit float: Enabled

      • Color: Metallic

    • _rough

      • 32-bit float: Enabled

      • Color: Roughness

    • _smooth

      • 32-bit float: Enabled

      • Color: Glossiness

    • _MS

      • 32-bit float: Enabled

      • Color: Metallic

      • Alpha: Glossiness

    • _MRO

      • Pack Individual RGB Channels: Enabled

      • 32-bit float: Enabled

      • Red Channel: Metallic

      • Green Channel: Roughness

      • Blue Channel: Ambient Occlusion

    • _ORM

      • Pack Individual RGB Channels: Enabled

      • 32-bit float: Enabled

      • Red Channel: Ambient Occlusion

      • Green Channel: Roughness

      • Blue Channel: Metallic

  • Once you have all of your Channel Packed Image settings finished, click on the “Save Config” button and save the setup as a Template so that you can Load the Config on subsequent projects.

1: Change to the Baking Tab in the PBR Painter main panel

2: Expand the General Bake Settings heading and untick “Include Resolution in File name” to cut down on the amount of text editing needed for renaming

3: Expand the Material Exporting heading

4: Change the resolution to 1K (from default 4K) and change the “Margin” to be 2px and “Bake Samples” to 64 → This will ensure a clean bake of the textures

5: Change the Channel Selection from “Automatic” to “Manual”

6: Tick the “Channel packing” box to set up for creating a template of Images to be baked for the texture set (similar to Substance’s system)

7: Click on the “+ Add channel-packed image” button to create an initial Image entry in the list

8: Using the + button, create 8 additional image entries - these will be renamed in the next part of this step

9: Rename list of Channel-packed Images to mirror above

10: Set the _colour Image 32-Bit Float to “True” and set the “Color:” to use the “Base Color” texture map in the bake sequence

11: Set the _ao Image “32-Bit Float” to “True” and set the “Color:” to use the “Ambient Occlusion” texture map in the bake sequence

12: Set the _normal Image “32-Bit Float” to “True” and set the “Color:” to use the “Normal” texture map in the bake sequence

13: Set the _metal Image “32-Bit Float” to “True” and set the “Color:” to use the “Metallic” texture map in the bake sequence

14: Set the _rough Image “32-Bit Float” to “True” and set the “Color:” to use the “Roughness” texture map in the bake sequence

15: Set the _smooth Image “32-Bit Float” to “True” and set the “Color:” to use the “Glossiness” texture map in the bake sequence

16: Set the _MS Image “32-Bit Float” to “True” and set the “Color:” to use the “Metallic” and the “Alpha:” to use the Glossiness texture maps combined together in the bake sequence into their respective channels of the output image

17: Set the _MRO Image “Pack Individual RGB Channnels” to True, “32-Bit Float” to “True”, Set the “Metallic” map to the Red Channel, Set the “Roughness” map to the Green Channel, and set the “Ambient Occlusion” map to the Blue Channel, and these will be combined together in the bake sequence into their respective channels of the output image

18: Set the _ORM Image “Pack Individual RGB Channels” to True, “32-Bit Float” to “True”, Set the “Ambient Occlusion” map to the Red Channel, Set the “Roughness” map to the Green Channel, and set the “Metallic” map to the Blue Channel, and these will be combined together in the bake sequence into their respective channels of the output image

Step 10: Run the Bake and manually save the Image Outputs OR Verify the bake outputs were auto-exported successfully

  • Before baking, rename the model you used for PBR Painter to be the name of the texture files (ie: Basin_Kidney)

    • When baking images and naming them, PBR Painter will use the name of the Bake mesh as the prefix and then add the suffix for the map type to the respective baked maps → So while this is against the typical naming conventions, this is specific to working with PBR Painter use models only and should not reflect naming for other assets

  • When everything is finished being configured, go ahead and hit the “Bake!” button

  • Depending on what you’ve done for your texturing and how detailed your model and texturing is, this process will take anywhere from a few seconds → a few minutes to finish baking

    • Note that if you are low on memory that its possible that the baking process will crash Blender due to running out of Memory. If this happens, adjust your Bake Samples number to a lower value until the bake is able to run.

      • With that said, most computers used by the Lab for Asset Production should be capable of running bakes with the settings created in the previous step

  • When the pop-up for “Bake Visible” comes up hit the “Ok” button

  • When your bakes have finished, switch back to the Layout tab and open the UV Editor

  • Use the Image Selection dropdown and scroll down until you see a number of entries that start with a 0 and have the same name as your mesh with the matching suffix for that map type → Select the _ao Map first

    • The 0 denotes that they are otherwise not being used and haven’t been set to a Fake User (dumby entry to save the texture as part of the Blend file)

  • In the Bottom Left Hand corner of the UV Editor window, Grab the Corner and drag to duplicate the pane so that you have 2 copies of the _ao map shown

  • Using the Image Selection drop down select the _colour map

  • Now duplicate the _colour map pane of the UV Editor Window (like you did to duplicate the _ao map pane for the _colour) until you have 9 panes total and you’ve set each baked image to the UV Editor panes in the sequence of

    • _ao

    • _colour

    • _metal

    • _MRO

    • _MS

    • _normal

    • _ORM

    • _rough

    • _smooth

  • As mentioned, when labelling the generated images through the Bake sequence, PBR Painter will take the name of the bake Mesh and use that as a prefix and then add a suffix for the particular bake type → when it does this, the baker will add a space in the label separating the prefix from the suffix: Remove the space to make sure the files can be read correctly

  • Now to save each image in the UV editor panes:

    • UV Editor Pane → Image dropdown → Save As

    • Navigate to where your Production files for the asset are being stored

    • Create a new folder and label this “Textures”

    • Inside the “Textures” folder, create 2 new folders and label them “Compressed” and “Original”

      • These are used as part of the overall asset production pipeline - Compressed is where the images that are run from Original through Caesium are stored

  • For each Image

    • Ensure File Format = PNG

    • Set Color to RGBA

    • Set Color Depth to 16

    • Set Compression to 0%

    • For all Images except for _colour set the Color Space to Non-Colour

Note on Texture Map Color Spaces
When saving each texture, be sure to pay attention to the Color Space setting for the image, as when it comes to PBR Texturing, the Color Space of the image is important to the way the rendering works. Its easy to remember:

_colour = sRGB

_ao = Non-Color
_normal = Non-Color
_metal = Non-Color
_rough = Non-Color
_smooth = Non-Color
_MS = Non-Color
_MRO = Non-Color
_ORM = Non-Color

Anything other than a Colour map (_colour, Base Color, etc) should be set to Non-Color unless otherwise needed

  • If using the Automatic Export setting, ensure that you verify the files saved correctly to their intended destination and you ensure to remove the Space in the file names
