VARLab : Adding Embedded Video for WebGL Deployments

Created by Stacey Dineen, last modified by User abfce on Aug 11, 2022

Currently, trying to play a video in WebGL is a limitation and if you do it by dragging a video in the video player object, all you will have is a black screen and wondering what happened. Luckily, we have a workaround to avoid this issue.

 Instructions

  1. Create inside the Assets folder a folder named StreamingAssets. It should be located normally where you clone your project (Project Name/Assets).

  2. Inside the StreamingAssets folder, add all your videos.

To get the videos to play, we need to get the videos via URL and play them by calling the play method of the Video Player. To be more precise:

  • Make sure to serialize the Video Player object:

  • To set the videos for the Video Player:

    • Application.streamingAssetsPath - this will point to the created StreamingAssets folder inside Assets.

    • Video name - make sure that the whole name is correct together with the file extension.

  • To play the videos, just call manually the play method of the VideoPlayer.
