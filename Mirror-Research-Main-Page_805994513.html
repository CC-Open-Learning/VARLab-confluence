Research

Basic Information

  • Mirror is based on Unity's abandoned UNET system. They took it and fixed it/added to it.

  • The version of Unity VARLab uses, which is 2021.3.24f1, is supported by Mirror.

  • To add Mirror to a project, you simply add it from the asset store.

  • Mirror is completely free and is also trusted by developers. It is even used in many games that have already been released. More can be read about those games here.

Topologies

This section will cover the various topologies that Mirror supports and does not support, as well as give a bit of detail on each. More resources can be found throughout the document if they are not easily understood.

Client-Server vs. P2P

Mirror, by default, is Client-Server. This means that ALL communication that happens must go through the server.

The biggest reason to use a Client-Server architecture is that it has centralization, meaning that there is a server in the “center” of all the clients, which all communication must go through. This can be seen as a pro or a con, depending on wants and needs. Furthermore, if the server is closed for some reason or crashes, the entire game will crash for all clients because it relies on the Server to function.

If you do not want this, you can use P2P (Peer to Peer) architecture, which is decentralized, meaning there is no center server. Rather, all clients have equal footing in the network. They can share resources, workloads, etc. If one of the clients helping act as a server crashes, the game will continue for everyone else. Only that player will be disconnected.

Each architecture has many other pros/cons, including security, scalability, cost, etc. More can be read about both architectures and the pros/cons of each here. Not being able to use P2P is unfortunate. It means that we will be stuck with the Client-Server architecture, not that it is bad, just that we can never change if we see a reason to use P2P.

Client Hosted vs. Dedicated Server

Mirror allows you to host a server in 2 ways. The first way is for one of the clients to host the game. This does not mean the client has any special privileges in the game. They are simply in charge of creating an instance of the game considered the “Server”. They then act as both a client of the game and the server.

The second way to host the game is with a Dedicated Server. This is where an instance of the game acts ONLY as the server. Mirror talks more in-depth about these and how they support them here.

Having the ability to choose between a client hosting a game and having a dedicated server is awesome! both are great options depending on needs, and we are not stuck using one method!
