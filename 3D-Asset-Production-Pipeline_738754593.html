CONCEPT STAGE

Step 1: Obtain Concept/Reference Art and Model Requirement Information

  • Ask Project Lead(s)/Asset Leads about Geometry requirements, Texture requirements, and any other particular information related to the Model (these should also be recorded on the Jira Ticket(s) themselves)

  • Ensure proper documentation of Sourcing when obtaining concept art from outside VAR Labs

  • When possible embed Concept Art/References on JIRA Ticket(s) along with links to where you got them from

Step 2: Whitebox Model based on Real World Dimensions to establish Proper Scale

  • Research Real-World dimensions of objects

  • Use Google/Wikipedia/The Usual Sources for information

  • If access to objects is available then take measurements yourself

    • Also be sure to take pictures to use as references

LOW POLY STAGE

Step 3: Model out Low Poly (L.P) Model

  • Try to utilize typical Low Poly modelling workflows and follow the mantra “Start Low, Finish Low”

  • Ensure good topology

  • Ensure good use of Body Modelling vs Greeble Modelling

    • IE: Keep mesh elements separate from eachother instead of trying to join their geometry’s

    • Ex: Joy Stick, Buttons, Speaker, and Coin Box mesh elements would be separate from the Body mesh of an Arcade Cabinet

  • No NGON’s. Period.

  • Avoid irregular Quads - cut them into Tris as follows:


Step 4: Apply Rotation and Scale

  • In Object Mode:

    • Select Object → Click on Object Tab → Apply → Rotation and Scale

  • Typically speaking, if a model’s Rotation and Scale are not at 1.000 (as often happens when scaling models in Object mode but not applying the Scales), it will strongly affect functions that are performed in Edit Mode (ex: when beveling, Bevels form strangely, UV maps look warped when unwrapping, etc)

    • TL;DR : If you see strangeness when modelling or performing functions, 9 times out of 10 it’s because the scales are off - perform the above Apply Rotation and Scale procedure to correct

      • If further issues persist then reach out to the Sr. Asset Developers for assistance (Ian or Topher)

Step 5: Separate L.P Mesh into Component Parts (each its own object)

  • In Object Mode:

    • Select All Objects in Object Mode → Edit Mode → Select All Geometry → Press P → Separate By Loose Parts

    • This is a good way to separate your mesh out into various objects so that you can view and hide each individually to make working in Edit Mode easier for things like Sharp and Seam Marking, UV Mapping, etc

  • Set up Object Child / Parent relationships as needed

Step 6: Go through each object and set up Sharps, Seams, Smooth Shading, Auto Smooth

  • Ensure Seams and Sharps are placed logically and appropriately on your model

  • To ensure less distortion in your UV Map, place your Seams along Surface Edges that bend (ie: A cube would have each face seamed as each face is a surface at a different angle)

  • Ensure Smooth Shading Enabled → In Object Mode:

    • Select All (A Key) → Right Click → Shade Smooth

  • Ensure Auto-Smooth is Enabled:

The following instructions are for Blender 3.0+ users

  • Go To Object Data Properties Tab → Normals → Auto Smooth → Set Degrees to 65 (some value adjustment may be needed)

  • Auto-Smooth works with Smooth Shading and ensures that Smooth Shading Normals appear cleanly - hard edges appear sharply and smooth surfaces appear cleanly smoothed.

The following instructions are for Blender 4.0+ users

  • Go to the Modifier Panel:

    • Add Modifier → Normals → Smooth by Angle

    • Adjust the angle from the default 30 and set to 65 degress → 65 is a strangely golden number for ensuring surfaces are smoothed correctly. Some further adjustment may be required.

    • If required, you can enable the Ignore Sharpness option to have the modifier ignore your Sharp edges (although not regularly recommended)

Step 7: Check Face Orientations (Normals) of Objects

  •  3D Viewport → Viewport Overlays tab → Face Orientation

    • See Screenshot below

  • All faces should appear Blue (aka: Normals are facing outwards)

  • If a face appears Red that shouldn’t be - In Object Mode:

    • Select Object → Edit Mode → Select Face (to flip) → Mesh tab → Normals → Flip

  • As a matter of good practice, when you’re modelling an object in Edit Mode, it is advisable to Select all Faces and Recalculate the Normals periodically

    • Select All Faces → Shift + N

Step 8: Ensure any Overlapping Verts are merged

  • In Object Mode:

    • Select Object → Edit Mode → Select All (A Key) → Enable Vertex Select Mode → Right Click → Merge Vertices → Merge By Distance (default is 0.0001)

    • Be careful to pay attention to your Vert Merges when you have needed verts that are close to eachother - adjusting the Merge By Distance value can cause unintended merges of such Verts

  • Make any needed adjustments to Normals / Sharps / Seams if they are changed by merging Vertices

Step 9: UV Map L.P Mesh Objects

  • Combine Body Mesh components UV’s onto as few UV Maps as needed - Coordinate with SR Asset Developers on the required #

  • Combine Greeble Mesh components onto as few UV Maps as needed - Coordinate with SR Asset Developers on the required #

  • Use ZenUV tool to Quadrify Islands (where applicable) and Stack all like-islands

    • NOTE: If ZenUV is not available, perform stacking and island squaring manually or with the TexTools addon

    • OnBoarding NOTE: Ensure getting a ZenUV License is done as part of first Month Onboarding for Blender Users

    • For External Vendors/Placement Students: ZenUV is not a required tool for you to have, it is used internally by VARLabs Asset Developers

  • Ensure that Like-Islands are stacked

    • Especially when texture information on that face is identical and reusable

  • Ensure each UV Map is Packed as squarely as possible and fills as much of the canvas as possible (achieving good Texel density)

  • Combine Component Mesh Objects together based on which UV Map they are using (Body, Greeble, etc) once UV Mapping is complete

ASSET REVIEW POINT #1:
Once you reach this point you should Export your UV mapped L.P Mesh as a FBX (non-Blender) or (if using Blender) save the Blend file and send to the Asset Dev Leads for an Asset Review (Ian or Topher)

Check for:

  • Accurate Dimensions

  • Applied Scales

  • Optimized Topology for High Poly work

  • Correct Normals

  • UV Mapping

    • Stacked Islands appropriately

    • Good Packing

    • Good Texel Space Usage

  • (if Blend file): Sharps, Seams, Smooth, Auto Smooth

HIGH POLY STAGE

Step 10: Duplicate and Combine L.P Mesh Objects (hiding the Originals), Duplicate the Duplicate L.P Mesh and use it to create the High Poly (H.P) Mesh

  • Blender uses the Subdivision Surface Modifier to quickly add additional geometry that Smooths out models (Similar to Turbo Smooth in other modelling programs)

  • Use Edge Loop Cuts to create Support Loops

  • Using Support loops try to get the H.P Mesh as close to the same shape as the L.P Mesh as possible (tightening up edge lines)

  • Try to avoid altering the base shape of the Mesh(s) OR adding additional geometry (so the mesh is not overly altered from the L.P Mesh)

  • If not using Blenders tools for Sculpting, export at this stage for use in ZBrush, etc

  • If sculpting in Blender, ensure that your Mesh has a high number of Poly’s so that there is ample surface data to work off

Step 11: Ensure H.P Mesh UV’s are aligned with L.P Mesh UV’s

  • Ensure that the L.P and H.P Mesh UV’s are aligned properly -

    • Correct UV Vertex Corners as necessary (primarily on the LP Mesh)

  • Misalignment of your HP and LP UVs is usually the cause of distortion in Normal/Ambient Occlusion bakes 

ASSET REVIEW POINT #2: Once you reach this point you should Export your H.P Mesh as a FBX (non-Blender) or (if using Blender) save the Blend file and send to the Asset Dev Leads for an Asset Review (Ian or Topher)

Check for:

  • Dimension

  • Scales

  • Topology is good for baking

  • Correct Normals

  • UV Mapping

    • UVs are well overlapped between the LP mesh and HP mesh

BAKING STAGE

Step 12: Setup Bake Cage

  • Ensure L.P Mesh and H.P Mesh are labelled properly (poly_[masterModel]_LP and poly_[masterModel]_HP)

  • Duplicate L.P Mesh and rename the Duplicate as poly_[masterModel]_bakecage

  • Select the Cage Mesh and in Edit Mode scale the faces out on their Normals so that the faces are just barely above the L.P/H.P Meshes

    • Scale on Normals: Alt+S

Step 13: Setup to Bake Normal/Ambient Occlusion Maps

  • Set Render Engine to Cycles

    • If Possible: Set Device to GPU Compute

      • Far faster render times than using CPU

    • Set Viewport: Noise Max Sample = 128

    • Set Render: Max Sample = 128

    • Set Render: Denoise

    • Set Performance: Memory to =

      • Use Tiling:

      • Tile Size = 128

  • Go to the Material Properties tab

    • Click the + button to create a new Material ID Slot

    • Click on “New” to generate a new material (Blender Default = Principled BSDF Node Setup)

    • Rename the New Material to be [masterModel]_BakeMAT

  • Open the Node Editor and setup Nodes for Baking as follows:

  • To add nodes:

    • Shift A → Click on Search → [Type in the Node you are looking for]

  • Create a new Image in each of the Image Texture Nodes and configure each image as follows:

    • Image Size: 4096x4096

      • Note: These images will be later downscaled in another step (512x512 or 1024x1024) - the initial large image size is to ensure clean baking results

      • Ambient Occlusion Image Name: [masterModel]_Bake_AO

      • Normal Image Name: [masterModel]_Bake_Normal 

      • Ensure both the Normal Image Node and AO Image Node Color Space are set to NON-COLOR

Note that it is generally advised to use textures that are no larger than 1024x1024 (Unity) or 4096x4096 (UE5 This follows the pattern of the “Power of 2” in images, which is a standardized set of image sizes that can be mathematically squared, these being: 8x8, 16x16, 32x32, 64x64, 128x128, 256x256, 512x512, 1024x1024, 2048x2048, 4096x4096, and so on.

Step 14: Normal Bake

  •  1: In the Node Editor, make sure that the Normal Map Image Texture Node is selected

  • 2: In Render Properties: Bake

    • Set Bake Type = Normal

    • Set Space = Tangent

    • Output Target = Image Textures

    • Output: Clear Image

    • Selected to Active:

    • Cage:

    • Cage Object: [Set to Cage Mesh]

    • Margin: Size = 4px

    • Margin: Adjacent Faces (where applicable)

  • 3: Run the Bake

  • 4: Ensure the Bake result is clean. Adjust + Troubleshoot the UV’s if any problems arise

    • Note: If distortions occur during the Normal Bake, any corrections made should also prevent distortions from occurring during the Ambient Occlusion bake

  • 5: Once bake is confirmed clean, ensure the Bake Image is saved through the UV/Image Editor

    • [From the UV Editor Window]: Image Tab → Save As → Set File Name = [masterModel]_bake_Normal → File Format = PNG → Color Depth = 16 → Compression = 0% → Color Space = Non-Color

Tip: Like with your AO Maps, if you are running into areas that are baking properly and ones that are not, consider saving the first bake, then duplicate your Normal mesh and separate the duplicate mesh out into the component parts that require a fixed bake, and then re-run your bakes on those parts. With the result, overlay it in Image Editing on the first bake you saved, and then combine the images by removing the un-needed space from the re-baked image

Step 15: Ambient Occlusion Bake

  • 1: In the Node Editor, make sure that the AO Image Texture Node is selected

  • 2: In Render Properties: Bake

    • Set Bake Type = Ambient Occlusion

    • Output Target = Image Textures

    • Output: Clear Image

    • Selected to Active:

    • Cage:

    • Cage Object: [Set to Cage Mesh]

    • Margin: Size = 4px

    • Margin: Adjacent Faces (where applicable)

  • 3: Run the Bake

  • 4: Ensure the Bake result is clean. Adjust + Troubleshoot the UV’s if any problems arise

  • 5: Once bake is confirmed clean, ensure the Bake Image is saved through the UV/Image Editor

    • [From the UV Editor Window]: Image Tab → Save As → Set File Name = [masterModel]_bake_AO → File Format = PNG → Color Depth = 16 → Compression = 0% → Color Space = Non-Color

Tip: Like with your Normal Maps, if you are running into areas that are baking properly and ones that are not, consider saving the first bake, then duplicate your AO mesh and separate the duplicate mesh out into the component parts that require a fixed bake, and then re-run your bakes on those parts. With the result, overlay it in Image Editing on the first bake you saved, and then combine the images by removing the un-needed space from the re-baked image

Step 16: Model Optimization

  • For most Static and Hard Surface Models you should be able to optimize the Geometry by about 40%-50% (Note: This does vary)

  • Generally speaking, you want to remove all unnecessary geometry from the model that doesn’t contribute to the model’s silhouette / visual fidelity and reduce the model to use as little geometry as possible so the model is lightweight for use in Game Engines. This can include (but is not limited to):

    • Removing extra edge loops

    • Merging vertices on non-quad faces

    • Removing edges that do not contribute to the silhouette

    • Etc

  • While optimizing the Geometry, pay attention to the Model UV’s, as removing/merging geometry can sometimes lead to UV warpage (although the warpage is usually easy to correct in the UV editor)

    • In Blender, be sure to enable the automated UV correction option when merging vertices and edges, as this will take care of a good chunk of your UV corrections for you

    • Make manual corrections as required

  • Ensure that any faces inside the mesh that are not required/viewable by the player are removed (backfaces, etc)

  • Avoid using Modifiers to optimize the model (Decimate, etc) - these tools often cause UV’s to become wrecked and require heavy fixing to correct. Rule of Thumb: Always optimize manually

  • Exact VAR Lab Optimization Standards are TBD

  • Geometry Target

  • Normals

SUBSTANCE STAGE

Step 17: Mesh Mask Setup

  • 1: Duplicate L.P Mesh and Rename → [masterModel]_surface_

    • The suffix space after surface_ is meant to allow for quick renaming of the separated mesh segments in this process 

  • 2: Go through the Mesh in Edit Mode and separate the Mesh out into Separate Objects based on the type of the surface

    • Gold Surface

    • Steel Surface

    • Etc

  • 3: Rename each separated Mesh Surface with a label that represents that surface

    • Example: [masterModel]_surface_gold

  • 4: Ensure that related Surface Meshes are joined together (EX: Gold Surfaces are joined together)

  • 5: Ensure that ALL Material information is removed from each Mask Mesh (removing all Materials and their ID slots)

    • [From the Material Panel]: Select Material Slot - Click on - Button

Step 18: Export Mesh Masks

  • To start Exporting: Select All Mesh Mask Objects → File → Export → FBX

  • In the Export Window ensure the following options are set:

    • Selected Objects:

    • Object Type: Mesh (Do NOT use any other option)

    • Transform: Apply Scalings = FBX Units Scale

    • Forward = -Z Forward

    • Up = Y Up

    • Apply Transform:

  • Be sure to name the Exported File to be the same as Mesh in the Blend File and then add a suffix of “_Substance” to the Export File name

    • Example: [masterModel]_Substance

Step 23: Texture Optimization

If you have your textures already at target resolution, negating the need for downscaling your images, then skip parts 1-5 of this step and proceed directly to part 6 (start of Caesium use)

  • 1: To start optimizing textures, first load the Texture Set(s) up in your Blender file in the Image Editor (_colour, _normal, _MRO / _colour, _ao, _metal, _rough, _normal).

  • 2: In the File Folder where the Texture Files are stored create a new folder and rename it “Originals”. Copy and Paste the Textures into this Folder so if any mistakes are made these can be reloaded

  • 3: In Blender open each Texture and resize the Images to either be 512x512 or 1024x1024

    • You will have to downscale each image one at a time

    • Check with Asset Dev Leads Ian and Topher for specific resolution requirements

    • In some RARE cases Textures can be 2048x2048 but this particular size NEEDS to be authorized by the Asset Dev Leads before implementing

    • Resize Procedure (in Blender Image Editor): Image Tab → Resize → [Set X and Y Dimensions]

  • 4: Save the Resized Texture when done.

    • Save Procedure (in Blender Image Editor): Image Tab → Save

Note: Blender Image Saves are generally not compressed and therefore need to be compressed with External Image Editors (Photoshop, Paint.NET, etc) HOWEVER Blender downscales USUALLY achieve the cleanest results (compared to downscaling in External Image Editors)

  • In other words: Always start by Downscaling in Blender and then compressing the Downscaled images in your image editor of choice

  • 5: Open each Blender downscaled Image in an external Image Editor and simply save the image and the Editor should apply its native compression algorithm to the Image

    • Note: Paint.NET achieves roughly 50%-70% better compression than Photoshop

  • 6: Open up Caesium Image Compressor to perform a final Image Compression. Follow the descriptions of the following screenshots for instructions

If the textures are showing heavy artifact-ing from the compression, re run the above process (texture optimization) and play around with the PNG Quality slider in Caesium until a good quality-compression ratio is achieved

Step 25: Unity Asset Import and Implementation

1: Open up the Unity File that the Model and Textures are for

  • If there is no particular Unity File to use then create a Blank Unity File, configure for use with URP (either the Standard Shader config OR with VAR Labs Proprietary MRO Shader) , and perform the rest of the procedures in this Step (25)

2: Navigate to where the Assets Folder is in the Unity File (create a new Assets master folder if using a Blank Unity file) and then create a new Folder and label it as the same as the Model being imported, minus any suffixes

  • Ex: sword_long_v1 would go in the folder “sword_long”

3: Drag and Drop the Exported Mesh and Texture files from your computer into the Unity Asset folder you just created

  • Configure the Import Settings where needed ← Check with Asset Dev Leads/Project Leads for any Specific Import Settings

4: Drag and Drop the Imported Mesh into a Unity Scene

5: In the Inspector set the Mesh’s XYZ Position Transform values to 0 (aka world center)

6: Set the Mesh’s XYZ Rotation Transform values 0, and then make any needed rotation adjustments to get the Mesh to be properly oriented

7: Drag the Mesh from the Unity Hierarchy Window into the Asset folder where the Mesh import is stored to make the Scene Mesh into a Prefab

  • Make the Prefab an Original one

  • Rename the Prefab in the Asset folder to be the same as the Mesh but instead replace any suffixes with _prefab

    • EX: weapon_sword_long_mesh would become weapon_sword_long_prefab

  • Delete the Mesh in the Unity scene when the above is finished as the scene prefab is no longer needed

8: In the Asset folder create a new material

  • Label it to match the textures being used with it and include a _MAT suffix

    • EX: weapon_sword_long_color would be used on weapon_sword_long_MAT

  • Set the Shader to be the one being used by the overall project ← See Project Lead(s)/Asset Dev Leads/Software Dev Leads for any specifics

9: Assign the Imported Textures to their related slots on the new material (based upon the required Shader’s requirements)

  • See Project Lead(s)/Asset Dev Leads for any specifics

  • Create additional Materials as required for the Mesh

10: Open the Prefab in the Asset folder and apply the Material(s) to the Prefab (either directly on the Mesh or through the Inspector)

11: Go back to the Unity Scene and bring the Prefab into the Scene and inspect it to ensure the Mesh displays properly

  • IE: No Normal issues, Mats appear properly, etc

  • If any issues exist, make any needed corrections or adjustments and then repeat the Export and Import steps as required

When all above steps are completed, either Save the Unity File according to the Project Requirements OR Export the Asset folder as a standalone Exported Asset Package

  • See Project Lead(s)/Asset Dev Leads/Software Dev Leads for any specifics
