VARLab : Network Identity

Created by Talon Ernst, last modified on Nov 13, 2023

The NetworkIdentity is a component you will need to add to any GameObject you want to spawn on and over the network. If you do not add it, the GameObject will not be able to be seen over the network. This component is meant to give it an identity over the network, hence the name.

It has the following inspector menu.

There is no need to change the “Asset Id”. It is automatically assigned. “Server Only” will only spawn the object the component is attached to on the server if it is checked, not the clients. Visibility can be changed
to either “ForceHidden” or “ForceShown” which will either hide or show the GameObject.

NOTE: In our hierarchy, if we have a parent GameObject with child GameObjects inside of it, only the parent GameObject should have a NetworkIdentity on it, the children can access it through code and can still be spawned on the network as long as their parent has a NetworkIdentity.
