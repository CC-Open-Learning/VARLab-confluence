1. EditMode Test

1.1. When to use it?

  • Use it to test code that runs only in the Unity Editor without launching the game.

1.2. What to test?

  • Editor-specific features: For example, check if custom Inspectors or Editor windows work properly.

  • Asset data: Verify if textures, audio files, etc., are imported correctly.

  • Example:
    "Check if this texture follows the compression settings correctly!"
    → This can be verified without running the game!

2. PlayMode Test

2.1. When to use it?

  • Use it when you want to test game logic while the game is running.

2.2. What to test?

  • Time-based tasks: For example, check if a character reaches a specific position after 2 seconds.

  • Example:
    "Check if the player jumps into the air when the jump button is pressed!"
    → This can only be verified while the game is running!

4. Code Coverage

4.1. What does it tell you?

  • It shows "which parts of the code were executed by the tests."
    In other words, it lets you check how much of the project’s code was tested by your tests.

  • Key Concepts:

    • Covered Code: Code that was executed by the tests.

    • Uncovered Code: Code that was not executed by the tests.

  • Example:
    "Check if my jump feature test executes all the jump-related code!"
    → If only part of the jump logic was executed, the remaining portion will be marked as uncovered code

4.2. Why is it important?

  1. Check test completeness:
    You can verify if all the codes in your tests are covered.
    → Example: "I can see if I’ve written enough test cases!"

  2. Find missing parts:
    Identify untested code and write additional tests for it.
    → Example: "Oh, I haven’t tested this conditional statement yet!"

  3. Ensure quality:
    The higher the test coverage, the more reliable the code becomes.
    → Example: "This code has been tested over 90%, so I can trust it!"

6. WebGL Build

6.1. In simple terms:

  • WebGL is a build target that allows Unity projects to run in web browsers.
    → "A feature that lets you upload your game to a website so anyone can play it!"

6.2. Key Features:

  • No installation required: Users can play the game by visiting the website.

  • Cross-platform: Works on most web browsers (e.g., Chrome, Firefox).

6.3. Prerequisites

6.3.1. Install WebGL Support Module for the Project

  • Find the project on Unity Hub

  • If WebGL is not on the list, please click the ‘Add platform.’

  • Install the ‘WebGL Build Support’ module if it is not installed for the Unity Editor version used in the project.

6.3.2. Build a WebGL

  • Build Setting can be found at ‘File >> Build Setting.’

  • Folders and files generated by the build

  • To run WebGL Build

py -m http.server 8000

The command py -m http.server 8000 is executed in a Windows environment where Python is installed.
