VARLab : Creating a Highpoly

Created by Jordan Robinson, last modified on Aug 09, 2021

The first step to creating a model for CORE is the highpoly, this is where we put in all our tiny details, nice chamfered/beveled corners, and just generally create a nice render of whatever it is we’re building.

This step is crucial as later when we perform our texture bakes we will be transferring the details from this model onto texture maps that can be used for our game-ready lowpoly model. This step is what gives us nice smooth normal maps and lots of detail in our ambient occlusion maps.

Highpoly Methods

There are many ways to create a highpoly model, this article will focus on the supporting edge loop method where we will be using extra supporting edges throughout our mesh to allow the smoothing/subdivision algorithm to provide the chamfered/beveled edge look that we desire.

This method is a great way to do hardsurface modelling as later on when we go to create our lowpoly model we can easily reduce the detail by disabling the smoothing modifier and removing our extra edge loops.

For simple objects we can save a copy before adding our extra supporting edges to skip a lot of the low poly work, however if any changes to the silhouette are made afterwards while making the highpoly, this old file will no longer match and you will need to take the time to reduce it.

Exercise

For this tutorial, we will be modelling a weblock, a piece of safety equipment to hold tensioned webbing for slackliners. In this case we will not be using blueprints or exact dimensions, rather we’ll base this model purely on reference photos.and eyeballed dimensions.

This shape presents a lot of challenges for a hardsurface modeller as there are many smooth surfaces with different bend radii on their edge chamfers. From a lowpoly standpoint it’s not the most difficult shape, but as a highpoly it presents numerous challenges that make it perfect for this example.

Preparing

For reference, this highpoly could take anywhere from 2-8 hours depending on which details the artist decides to model, their experience level, and workflow efficiency.

Before you start, it’s always smart to check online for reference images for more angles of your subject.

If you don’t already have an excellent set of hotkeys, use this exercise to develop your hotkey collection and really get used to them. The amount of time you can save over using the UI is worth the effort to develop your own hotkey layout.

As a personal example, I can use Orbit, Pan, Vertex/Edge/Poly/Element Select, Move, Scale, Rotate, Extrude, Connect, Bridge, Select Ring/Loop, Remove Edge, and many more tools with only my left hand due to my hotkey setup.

Let’s Model

We’ll start by blocking out the model and getting the general dimensions correct. As the model is perfectly symmetrical we can use a Symmetry modifier so that we only need to edit one side. While you can attack this model any way you’d like, we’re going to start with the long side.

Immediately we can see two instances of circular geometry so it’s probably best to start out with the caps of two cylinders.

We want these to have a reasonable amount of sides since later on when we reduce our highpoly we don’t want the lowpoly to be very jagged.

From here we’re going to extrude these edges to get a ring around our cylinders.

With this we can delete the centers and start working on creating the rest of the front geometry.

After doing a bit more modelling I realized we need to add another hole for the pin that we’ll be making later.

After spending a bit more time blocking out the silhouette I’ve run into my first major geometric issue that will prevent the highpoly from smoothing well.

To remedy this, I’ll remove those three edges and add extra vertices to return the geometry to clean quads around the edges.

A bit more poly modelling later and we’re starting to see the general silhouette of the side. You can see there’s generally an effort being made to make sure the edges of my model that need to have nice chamfered edges are solely composed of quads.

We try to have triangles only where the model is flat so that they won’t impact the subdivision algorithm negatively later on.

In this geometry we can see a few areas with topology that won’t play nicely with subdivision later on. These areas will need to be converted to nice quads and have the triangles converge where the model is flat rather than at an edge where the silhouette needs to make a bend.

To add some support loops to the areas I know will get smoothed together later, I’m using a combination of Select Ring and Connect to select existing edges around my model and add new edge loops between them.

For areas that don’t play nicely with those tools, I’ll use the Cut tool to manually add in edge loops where I know they’ll be needed.

We can see the problem areas I identified before have been dealt with to provide a nice smooth edge for the small rise between the front and rear sections of this side plate.

Now I can start extruding this face and begin on the connecting parts in between the two sides.

We’re starting to get there now, the two sides have been connected nicely and we can add the middle diverter piece before adding the inside faces and finishing up the basic geometry.

Now the only thing left is to create the inside faces.

You can see that there is not much concern for how ugly the topology is as long as the ugly topology takes place on the flat surfaces of our model. Those issues will not harm our smoothed highpoly as long as they occur in a flat section.

To finish up this part of our highpoly, we need to add in the supporting edges for the rest of the model before we can add our Turbosmooth modifier.

The red arrows show where we can see that we need some support loops, otherwise this area will smooth into an oval shape.

After much time spent adding support loops to the edges using Connect, cleaning up vertices with Weld and using Remove on improper edges, we’re finally finished our support loop highpoly.

Smoothing

Now we can bask in the glory of our highpoly and see if we made any major mistakes as we add our Turbosmooth modifier to subdivide the mesh. For programs other than 3DSMax, instead of Turbosmooth it will be whatever geometry subdivision algorithm your program uses.

Recessed Areas

Now this looks pretty good but what about the recessed sections? We don’t actually need to add them to our highpoly, we can model them on top instead to keep things simple.

We can start by modelling the recessed shape as a plane on top of where we want it.

Now using Inset we’ll pull the edges inwards to create the support loops for the top section of this recessed area.

After extruding this inwards we add another set of supporting loops using Inset again and we’re good to go.

After applying the Turbosmooth/Subdivision to this recessed area, we can see that it appears to be going into the model even though it’s outside and on top of it.

This trick is very useful for baking textures as this portion will appear to be recessed on our normal and occlusion maps when we bake them.

After repeating that process for the small indented area where the cable comes out, we’re pretty much done the main weblock assembly. Now we just need to start on the pin.

The Pin

For creating the pin, there’s not much to it, we’ll stack up some cylinders with supporting edges and call it a day. We’ll start with an 8-sided cylinder.

Now we can stack up some more cylinders on it, making sure to give the handle of the pin a few more sides as it’s going to be larger and more visible.

With these finished and the support loops added, we can toss on Turbosmooth and see how it looks.

Looking pretty good! With this we can make any final touches we want to the support loops to influence our edge radii and begin with our lowpoly.
