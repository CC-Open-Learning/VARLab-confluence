VARLab : Project Research and Ideas

Created by Nick Harrison, last modified by Ryan Wallace on Sept 19, 2023

An onboarding project that will allow new team members to create minigames to be part of a carnival-style environment. Each mini-game will be accessible through the main scene but, when entered, will transition into a new mini-game-specific scene.

Possible ideas for how the carnival setup could look:

  • Space theme travel through portals to different minigames

  • Traditional carnival theme travel into tents for the mini-game

  • Fall fair theme to travel between booths for mini-games

Every mini-game could have some entrance or front that will be viewable/enterable from the main scene. A player could walk down the main scene and look at each mini-game entrance. This would mean that in the main scene, there should be a customizable prefab for a mini-game front where new developers can customize the prefab to have their minigame names and mini-game specific details/look/colour/etc.

Project Requirements

  • main scene for exploring and connecting to other scenes

  • a character controller to move through the environment

  • a way to transition between scenes

Design Choices - Project Look/Style/Feel

Potentially, a low poly look/feel for this project is beneficial in terms of resources and helps make the game easier to run over a resource-intensive, more realistic look.

There is not a large set of fair/circus-related free assets, which might be a challenge - low poly assets might be easier to find or create.

Example low poly asset: https://assetstore.unity.com/packages/3d/props/tdg-italian-ice-cart-257881

Fair Style (Fall fair feel, rides, booths)

A fantastic asset is available on the Unity store (however, it is not free (it is $49). This asset pack would give us modular lighting, fair-style rides, stands, food trucks, cars, signs, arcade games, indoor arcade, slushy machines and more (essentially everything you would find at a fair) where we can assemble a fair-style world that can lead to each mini-game scene. The package has 350 prefab assets.

This asset package also includes background scenery to support the whole fair environment (night sky, mountains, Ferris wheel)

https://assetstore.unity.com/packages/3d/props/tdg-fairgrounds-low-poly-3d-pack-241846 { good fairgrounds package including lots of options for a fair setup

The asset package is created by Theory Digital Games, and they have a video playlist showcasing some of the assets:

https://www.youtube.com/playlist?list=PLCQlJ0y66hGSPhWxrMT73aVFYjlATZ87a

This package would allow us to have booths for each mini-game, for example:

We would need to add the functionality to make these booths act as a way for users of the main fair to enter into the mini-game scene. A good suggestion is we could have example minigames in place as well for new members to use as a reference for setting up their games (maybe a mini-game for balloon popping.)

We would also need to find a way to make the booth modular so that members can change the name/colour/and look of their booths if we were to choose the fair-style setup.

This package already offers the ability to change posters with PNGs, so we can easily customize assets. There also are included animations, scripts, and light shows so we can modify the fair to look the way that we would like. This set of prefabs for the fair means that we would be able to expand the fair as much as we would like to.

Lastly, this package wouldn't need to only be used in the main scene. This package will offer tons of functionality into each mini-game as game assets for different fair activities are also included (e.g. go-carts and food courts). There is also a full set of prizes as assets that can be won from potential mini-games if we or future developers choose to add a prize component to their mini-games.

This package gets updates with new models as well.

The package includes scripts to support a fairground feels (i.e., moving Ferris wheel)

Functionality Ideas (the tech stuff)

requirements

  • character controller

  • teleportation

  • main scene environment

  • ability to add new booths/planets/tents/whatever we choose to be the expandable areas

Character Controller

  • player can move

  • player can jump

  • player can interact with obj

  • VR movement?

Teleportation

this is just a list of ideas for the teleportation:

  • a consistent object the player interacts with to teleport to another scene

  • maybe a animation of entering something or doing something with the teleport obj before loading the next scene

  • a prefab to always allow the dev to decide what scene this obj teleports to example public variables to be set in the inspector

Main Scene Hub World

this is just a list of ideas for the main hub world:

  • at least 4 buildings set up

  • a character to introduce the world upon starting the game this can be skippable with a UI element

  • colliders that interact appropriately with the player

New Onboarding Members Spaces

  • Booth templates

  • Sign templates

  • Spaces to add new areas accessible from the main map

  • Mini-game usable objects that new members can use in their minigames

  • Members can choose where to place their mini-game in the main scene by selecting an expandable area on the main map.

    • Could setup greyed out boxes where members can add their mini-game to

    • Birds eye map view of main world to select spaces to add new mini-game

    • This would be a collaborative area where new members can experience merge conflicts and working on a single project as a group

New Onboarding Process Discussion

  • An existing member should manually add as many blank booths/tents/planets (whichever we decide is best for the users to present their mini-game in the main map)

  • Each booth should be named and assigned to new members. (The mini-game representation name can be the new team members' name until the new team members rename it the name of their mini-game.)

  • New team members can then join in and modify their mini-games representation on the map

    • This process will require new members to experience a large code base and work on their own space without disrupting the existing spaces

    • New team members will need to manage merge conflicts while working through this process

    • Each mini-game representation on the map will act as a way of entering that mini-games scene
