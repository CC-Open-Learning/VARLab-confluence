VARLab : Upload DLS to VARLAB-DEV1

Created by Stacey Dineen on Jan 20, 2022

If you are currently working as a co-op or work study placement at VARLab, you cannot upload content to VARLAB-DEV1 for security reasons. Ask a non-coop team member to take over this process at Step 3. If you aren’t sure who to ask, start with Stacey or Jordan.

  1. Build and Name: Start by building or exporting your DLS to a SCORM package. You can name the package whatever you want, but the filename must include the DATE. The SCORM package is a zip file - if it doesn’t have a nice name on export, just rename it in your local drive before uploading.

  2. Archive to SharePoint: Upload your SCORM package to the appropriate folder in your SharePoint project workspace. If this is the first SCORM build for this project, simply create a folder called SCORM. This is an important step because SCORM packages are not stored in our code repositories, and cannot be downloaded from the LMS (eConestoga). SharePoint is our SCORM repo.

  3. eConestoga VARLAB-DEV1: Navigate to the VARLAB-DEV1 course on eConestoga. You must login with your staff account to access this course. Click on Content in the upper left corner, then browse the Table of Contents to find your DLS project folder. If a folder does not exist (look carefully, it probably does) you will need to create it by clicking on ‘Add a module’ at the bottom of the list.

  4. Upload/Create: From inside your DLS project folder, click on the Upload/Create button and then select New SCORM/xAPI Object from the dropdown list. This will open the Add Course Package - SCORM/xAPI dialogue. Click on ‘Bulk upload’ in the upper right corner of this dialogue and then select your newly built SCORM package from your local drive. This takes awhile, there is a progress bar to let you know how it’s going.

  5. Configure Course Package Properties: You can edit the title of your SCORM package here, provided you keep the date as part of the title. Everyone prefers a different date format, it seems, so just make sure it is clear (ie. don’t give me 04-05-22 so that I can’t tell if this build was from Apr or May! Try 04MAY2022 or anything that removes ambiguity). Keep the default settings for ‘Use embedded player’ and ‘Use built-in navigation’. Click ‘Next’ to be returned to the Add Course Package - SCORM/xAPI dialogue. Note: The UI is weird and might be cut off so that you cannot see the buttons at the bottom. Grab and drag the dialog around on your screen to access the Next button, if it isn’t visible.

  6. Adding Course Package: Your newly uploaded SCORM package will now appear in the list. Select it and click Next to open the ‘Adding Course Package’ dialogue. In this box, you must change the default value for ‘Do you want a grade item created for this instance?’ to be NO. Keep the default values for Version Control (always display latest version) and Course Package Player Options (Use embedded player). Click ‘Add’.

  7. Verify the New Package and Hide the Old Package: When your new SCORM build is successfully added to the module, click on it to launch it (this may occur automatically). This is a great time to check and make sure that you built what you intended to build (e.g. didn’t accidentally include all CORE projects, build was successful, etc). You don’t have to play the whole thing right now, but you do need to verify that your build is successful and contains the sim that you wanted to publish. Once you have launched and verified your new build, you should hide the previous build to avoid any confusion during playtesting. Click on the down arrow beside the title of your previous build title and select ‘Hide from Users’ in the dropdown. Grab a link to your new package, you’ll need it for your demo!
