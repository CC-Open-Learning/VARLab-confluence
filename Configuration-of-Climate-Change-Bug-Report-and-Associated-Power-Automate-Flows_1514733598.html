Overview

The Climate Change Bug Report is a SharePoint list configured to automatically generate a unique Bug ID for each new issue created. This Bug ID is formatted as a combination of “BUG-CurrentDateTime-SharePointItemID,” where the SharePointItemID is an auto-incrementing value.

The Bug ID is a single-line text column where data is generated using the Power Automate workflow "Climate Change - When a new item is added in SharePoint, create Bug ID and Item Link".

The Item Link is a hyperlink column in the SharePoint list, automatically populated with the item's display link once the workflow is executed.

Note: After creating a new bug entry in the list, it may take up to one minute for the Bug ID and Item Link to be generated. If the Bug ID does not appear immediately after creating the entry, please wait for a minute and refresh the page. The Bug ID and Item Link should then be visible.

Climate Change Bug Report - Required Fields

When adding a new item to the list, the following columns are designated as mandatory according to the requirements. We can modify these mandatory columns at any time based on changing requirements.

Climate Change - When a new item is added in SharePoint, create Bug ID and Item Link - Power Automate Flow Steps

This workflow is triggered whenever a new item is added to the SharePoint list “Climate Change - Bug Report.”

Once triggered, the workflow retrieves the newly created item and updates the Bug ID and Item Link columns accordingly.

Climate Change - Export attachments from SP Bug list to shared location in SharePoint - Power Automate Flow Steps

“Climate Change - Export attachments from SP Bug list to shared location in SharePoint” is a power automate flow that needs to be run on demand. It is triggered manually. Whenever necessary, go to the workflow and click "Run."

This flow is used to retrieve all attachments from the “Climate Change - Bug Report” SharePoint list and save them to a shared location. When triggered, the flow will retrieve all attachments created in the past 30 days and save them to a different location in SharePoint.

Note: As we currently do not have a service account, the workflow is owned by Neethu and shared only with Cristina and Dayne to minimize risks. Any incorrect changes to the workflow could potentially disrupt the entire process.
