Initial Set-Up

The MRO Universal ShaderGraph can be found on SharePoint here if it’s not already in your project. Note that this is still a ShaderGraph at this point and has not been compiled down into a Shader script yet. As the MRO Triple Mask Shader Graph is still being developed it will stay as a ShaderGraph for the foreseeable future. Don’t worry though - Unity will handle packing this down at build time!

Once your project has the MRO triple Mask Shader Graph added to it, setting up a material should be fairly straightforward.

Lets begin by creating a Material. You can do this by by right-clicking on your project folder and selecting Create> Material

Once you have a material created, give it a descriptive name, then take a look at the Inspector

This is Unity’s default lit material, and its perfectly servicable, but does not support an MRO workflow, and is lacking some features that the MRO Triple Mask Shader has built-in. Let's change the shader we want to use from the drop down labeled “Shader” near the top of the inspector. Navigating all the Shaders that are in Unity can be a bit confusing, so I recommend just searching “triple” to get the shader we want

We should see the Inspector update to reflect the newly selected shader. In Figure 04 I have collapsed all the Surface Inputs options, though these will likely be expanded by default.

Now that you have a material that's using the MRO Triple Mask Shader, lets break down the sections and what they do.

Surface Options

First up is the Surface Options. These are Unity default settings that most shaders should have access to. I won’t go too in depth here, but will draw your attention tot he Surface Type parameter, where we can set this to either Opaque or Transparent. If we want to use any of the Transparency features below, we need to make sure we have this setting set to Transparent.

You may also have need of the Render Face Setting on a by-case basis.

Of course, all of these settings will affect what the material looks like in game, but we most care about the transparency settings.

Surface Inputs

Colour

This section is where we set the colour or albedo information of the shader.

Color Map: This is where the image representing the colour information will be socketed.

Color Map Tiling: This allows us to change the tiling of a colour map, meaning we can make the texture appear smaller and tile more often by increasing the value here, or make the image larger and tile less frequently by decreasing the value here. You can change the X and Y values separately.

Color Map Offset: This setting allows us to nudge the map around to change where the texture falls on the model. This is most useful for tiled textures, but may also be useful with simple atlassed texture variations.

Transparency

This section allows you the change the intensity of a transparency, but also the clipping on what is considered transparent.

Note that there is no transparency map - this information is taken from the alpha channel of the Color Map socketed into the Colour section.

Transparency Range: This setting allows you to determine what values in the greyscale image are considered transparent

Transparency Fuzziness: This setting allows you to blur the edge of a transparent section. It works best if there is already some amount of gradient between the opaque and transparent sections, though works either way.

Transparency Intensity: This setting allows you to adjust just how transparent the entire material is

Transparency Threshold: This setting allows you to clip the alpha map so there is a hard cut-off at a specific greyscale value.

Tint

The tint section allows us to isolate areas of the material with a special map and change the colour of that area, either as a solid fill, or as a tint. This can be used to create colour variations of objects, and may be able to be leveraged to customize a player’s avatar and their clothing or accessories in the future.

Most of the settings below are repeated 3 times, once for red, green, and blue.

Three Channel Tint Mask: This is a special map that is able to isolate 3 different areas on a texture to be able to be tinted separately. Each of the three masks is stored in a colour channel of the image

Has ___ Tint: This is a toggle to set if the corresponding channel mask is being used or no. If this is false, none of the other settings for this colour channel will have any effect on the material.

___ Channel Tint Colour: This setting lets us choose what solid colour will be pushed into the areas that has been masked out.

Invert ___ Channel Mask: This is a toggle that will invert which areas are affect by the mask. If this is true, the mask will read the black area instead of the white area.

___ Channel Mask Range: This setting allows you to determine what values in the greyscale image are considered to be part of the mask.

___ Channel Mask Fuzziness: This setting allows you to blur the edge of a mask. It works best if there is already some amount of gradient between the opaque and transparent sections, though works either way.

Preserve Black - ___ Channel Mask: Looks for black areas on the base textures and puts them back on top of the tinted area. This is useful for areas with black text.

Normals

The Normals sections allows you to make adjustments to the normal map.

Normal Map: This is where the image representing the information for the normals is socketed.

Normal Intensity: This allows you to change how much the normal map is interacting with the lighting system, making a surface either more or less bumpy. This value defaults to 0.5, which is 50%. This was done intentionally as Unity likes to be fairly heavy handed with its interpretation of a normal map. If detail is lost, simply dial up this setting

Normal Map Tiling: This allows us to change the tiling of a normal map, meaning we can make the texture appear smaller and tile more often by increasing the value here, or make the image larger and tile less frequently by decreasing the value here. You can change the X and Y values separately.

Normal Map Offset: This setting allows us to nudge the map around to change where the texture falls on the model. This is most useful for tiled textures, but may also be useful with simple atlassed texture variations.

Metallic, Roughness, Occlusion

This MRO section allows us to change the influence of the metallic, roughness, and occlusion maps.

Most of the settings below are repeated 3 times, once for metal, rough and ambient occlusion.

MRO Map: This is where the image representing the information for the metallic, roughness, and occlusion is socketed.

___ Intensity: This allows you to dial up or down how metallic/rough/in shadow the entire material is.

___ Tiling: This allows us to change the tiling of a map, meaning we can make the texture appear smaller and tile more often by increasing the value here, or make the image larger and tile less frequently by decreasing the value here. You can change the X and Y values separately.

___ Offset: This setting allows us to nudge the map around to change where the texture falls on the model. This is most useful for tiled textures, but may also be useful with simple atlassed texture variations.

Emission

This section allows you to configure areas of the model that glow using an emission map.

Is Emissive: Lets you turn the emission on or off. If its false, none of the other settings below will have any effect until it’s made true.

Emissive Map: This is where the image representing the information for the emission is socketed.

Emission Tiling: This allows us to change the tiling of the emission map, meaning we can make the texture appear smaller and tile more often by increasing the value here, or make the image larger and tile less frequently by decreasing the value here. You can change the X and Y values separately.

Emission Offset: This setting allows us to nudge the map around to change where the texture falls on the model. This is most useful for tiled textures, but may also be useful with simple atlassed texture variations. ((Animating this parameter via script can make some pretty darn cool effects ))

Use Mask: Lets you use a mask to occlude areas from the emission. If this is false, none of the mask settings below will have any effect on the material until its made true

Emissive Mask: This is where the image representing the information for the mask is socketed. This information is taken from the alpha channel of whatever image is plugged in.

Emission Range: This setting allows you to determine what values in the greyscale image are considered part of the Emissive Mask.

Emission Fuzziness: This setting allows you to blur the edge of the Emission Mask. It works best if there is already some amount of gradient between the opaque and transparent sections, though works either way.

Rim Lighting

This section allows you to configure a rim light. It uses the Fresnel Effect to calculate this. I don't want to get into the physics of light here, so play with this one to understand it better!

Has Rim Lighting: Lets you toggle if there is rim lighting or not. If this is false, none of the other settings will have any effect until it’s made true.

Rim Colour: Lets you set the colour for the rim lighting

Rim Power: This setting lets you set the power of the rim light this is effectively the sharpness of the rim light. Higher numbers mean the lighting will creep LESS.

Rim Level: This setting lets you adjust the level of the rim light. Effectively, this means just how obvious the rim lighting is.
