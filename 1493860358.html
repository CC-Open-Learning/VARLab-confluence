Improved Process for Tracking Tester Feedback and Testing Time

As part of our process improvement, we have implemented a system using an MS Form questionnaire to track each tester's testing time and feedback. This questionnaire simplifies data collection and ensures an organized workflow.

The improvement process was implemented in three stages. At the end of each stage, we evaluated the effectiveness of the changes made. Based on these observations, we refined and enhanced the process. This iterative approach allowed us to continuously improve the process over time.

Stage

Tool / Approach

Challenges Faced

Actions Taken

Stage 1

Polly Questionnaire

  • Inconsistent response rates.

  • Testers had to enter time in a rigid format, making it inefficient.

  • Paid subscription required for more than 200 responses.

  • Decided to explore alternative tools due to scalability and efficiency issues. ( Stage 2)

Stage 2

Forms Application

  • Questions like “How many defects have you raised so far?” led testers to think defect reporting was mandatory.

  • Resulted in duplicate defects in the backlog.

  • Introduced branching logic to improve relevance.

  • Rearranged questions for better clarity and flow. ( Stage 3 )

Stage 3 ( Current )

Enhanced Forms Application with Improvements

We haven’t received any concerns from the Team leads so far.

Workflow Overview

Questionnaire and Data Capture:
Testers fill out the MS Form questionnaire during their testing sessions. Once submitted, the responses are automatically logged into a SharePoint list, which serves as the centralized repository for all feedback and testing data.

  • Power Automate workflow (Cross Team Testing - Record MS Form responses in SharePoint) will get auto triggered when a response is captured in the MS Form.

Note: The Power Automate link will only work for the assigned owners and co-owners.

Data Analysis and Evaluation:
The collected data will be analyzed and shared with the Principal Investigators for further evaluation and decision-making.

Steps to Follow

Assign Applications to Testers:

  • Stacey Dineen (Unlicensed) or Cam Turner will share the divided application list to be tested with the team via the group chat.

Distribute the MS Form:

  • Share the MS Form questionnaire with the testers.

  • Set the end date for submitting feedback to Monday at 11 AM to ensure sufficient time for all testers to complete their entries.


Prepare for the Next Testing Round:

  • Before the next cross-team playtesting session, an automated email will be sent to the email addresses of the individuals responsible for managing the MS Form.

  • Power Automate workflow (Cross Team Testing MS Form - Delete old responses and set a new end date) is scheduled to run every 2 weeks on Thursday.

Note: The Power Automate link will only work for the assigned owners and co-owners.

  • The email will serve as a reminder to clear the response sheet and set the end date so that the questionnaire can be reused for the next round.

Limitations :

  • A SharePoint list should be associated with a dedicated service account.

  • Deleting previous responses from the form is managed manually through reminder emails.
