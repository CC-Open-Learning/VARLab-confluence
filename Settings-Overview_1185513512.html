VARLab : Settings Overview

Created by User 96d6e, last modified by estronge on Jul 10, 2024

This page will give an overview of the settings that are throughout the software process, including Camera Alignment Settings, Build Model and Texture and more.

It’s important to note that it sometimes take a few different tries to find the right settings to use for a specific model. We are working on creating a detailed overview of the settings related to complex vs simple geometries and textures so that the estimation of the settings needed will be close to perfect.

The process settings table with notes is found below.

Process Settings

Process Step

Settings Explanation

Align Photos

General Settings

Accuracy: Determines the precision of the image alignment.

  • Highest: Maximum accuracy, but requires the most processing time and memory.

  • High: A good balance between accuracy and processing time.

  • Medium: Suitable for large datasets where processing time is a concern.

  • Low: Faster processing with less accuracy.

  • Lowest: Quickest processing, minimal accuracy.

Generic Preselection: Uses image metadata (such as image size, date, and other EXIF data) to reduce the number of photo pairs that need to be matched, speeding up the alignment process.

  • Improves the efficiency of the photo alignment process by pre-filtering images that are not likely to overlap significantly, thus reducing the computational load.

Reference Preselection Dropdown (Estimated, Disabled): Uses GPS data or other reference data if available to help with the initial alignment, improving accuracy and speed.

  • Enhances the alignment process by using spatial information to make educated guesses about which images are likely to overlap. If disabled all image pairs are considered for matching which could slow down the process.

Reset Current Alignment: If selected, the current alignment will be discarded, and the photos will be aligned from scratch. If not selected, the software tries to improve the existing alignment.

Advanced Settings

Key Point Limit: The maximum number of key points to detect on each image.

  • Key points are unique features in the photos that the software uses to align the images. A higher limit increases accuracy but also processing time and memory usage.

Tie Point Limit: The maximum number of tie points to use for alignment. Tie points are points that are visible in multiple images and are used to align the photos relative to each other. A higher limit improves accuracy but requires more processing time and memory.

Apply Masks To

Guided Image Matching: Uses masks to ignore certain areas of the images during the matching process, which can help in cases where there are repetitive patterns or unwanted objects.

Adaptive Camera Model Fitting: Adjusts the camera model based on the masks to improve the accuracy of the alignment, especially useful for complex or distorted images.

**When you hit start you can expand details and see which photos are not gathering as many points (quality check)

Build Model

General Settings

Source Data

  • Dense Cloud: Uses the dense point cloud generated in the previous step.

  • Depth Maps: Uses depth maps for creating the mesh, which can be faster and use less memory.

Surface Type

  • Arbitrary (3D): Suitable for most general-purpose 3D models.

  • Height Field (2.5D): Optimized for terrain and other surfaces that can be represented as height fields.

Quality

  • Ultra High, High, Medium, Low, Lowest: These settings determine the level of detail in the depth maps or dense cloud used for generating the mesh. Higher quality settings result in more detailed models but require more processing time and memory.

Face Count

  • Custom: Allows you to specify the desired number of faces in the mesh.

  • High: Generates a high number of faces, capturing more detail.

  • Medium: Balanced face count and detail.

  • Low: Generates fewer faces, faster processing, less detail.

Advanced Settings

Interpolation

  • Enabled: Interpolates between points to create a smoother mesh.

  • Disabled: No interpolation, resulting in a more rugged mesh.

  • Extrapolated: Allows the mesh to extend beyond the dense cloud boundaries.

Depth Filtering

  • Disabled: No depth filtering applied, can result in noisy point clouds.

  • Mild: Removes a small amount of noise, preserves more details.

  • Moderate: Balanced noise removal, suitable for most projects.

  • Aggressive: Removes a significant amount of noise, might lose some details.

Calculate Vertex Colors

  • Enabled: Colours the vertices of the mesh using the colours from the original photos.

    • This can improve the visual appearance of the mesh.

  • Disabled: No vertex colouring.

Use Strict Volumetric Masks

  • Enabled: Uses strict volumetric masks to constrain the volume of the reconstructed model. 

    • This can help improve the accuracy of the model by excluding areas outside the defined volume.

  • Disabled: Does not use strict volumetric masks.

Reuse Depth Maps

  • Enabled: Reuses previously calculated depth maps from earlier processing steps.

    • This can save time by avoiding recalculating depth maps.

  • Disabled: Depth maps are recalculated, which can increase processing time but may improve model accuracy.

Replace Default Model

  • Enabled: The newly built model will replace the default model in the project.

  • Disabled: The newly built model will be added as a separate model, allowing you to compare different versions.

Build Texture

General Settings

Texture Type

  • Diffuse Map: Standard texture map that defines the color of the surface.

  • Occlusion Map: Texture that simulates the effect of ambient occlusion, adding shadows to the crevices and cavities to enhance realism.

  • Normal Map: Texture that simulates the appearance of small surface details without adding geometry.

  • Specular Map: Texture that defines the shininess and highlight of the surface.

  • Roughness Map: Texture that defines how rough or smooth the surface appears.

  • Metallic Map: Texture that defines the metallic properties of the surface.

Source Data

  • Images : Uses the original images to generate the texture.

  • Orthomosaic: Uses the orthomosaic generated from the photos for texturing.

    • This uses a stitched-together image from all your photos to create the texture.

Mapping Mode

  • Generic: Suitable for most models, distributes texture evenly.

  • Adaptive Orthophoto: Optimized for creating orthophotos, adapts to the terrain.

    • This adjusts the texture to fit the shape of the ground.

  • Orthophoto: Creates textures suitable for orthophotos.

    • This makes textures that work well for top-down images of the area.

  • Single Photo: Uses a single photo for texture generation.

  • Keep UV: Uses existing UV mapping.

Blending Mode

  • Mosaic (default) : Blends textures from multiple images, prioritizes quality.

  • Average: Averages textures from multiple images, can reduce visual quality.

  • Max Intensity: Uses the highest intensity value for each pixel, useful in low-light conditions.

  • Min Intensity: Uses the lowest intensity value for each pixel.

Texture Size / Count

  • Custom: Allows you to specify the size and number of textures.

  • Optimal: Automatically selects the optimal texture size and count based on the model.

Enable Hole Filling

  • Enabled: Fills small holes in the texture to create a more seamless appearance.

    • Note: It does it’s best

  • Disabled: Leaves holes in the texture, which might be necessary for certain types of analysis.

Enable Ghosting Filter

  • Enabled: Reduces the effect of ghosting, which can occur when moving objects are captured in different positions in different photos.

  • Disabled: No ghosting filter is applied.

Transfer Texture

  • Enabled: Transfers texture from an existing model or a previously generated texture map.

  • Disabled: Does not transfer existing texture information.

Anti-Aliasing

  • Enabled: Applies anti-aliasing to the texture to reduce the jagged edges and create a smoother appearance.

  • Disabled: No anti-aliasing is applied, which might result in a sharper but more pixelated texture.

Build Point Cloud

General Settings

Quality

  • Ultra High: Generates the densest and most accurate point cloud but requires the most processing time and memory.

  • High: Good balance between detail and processing time.

  • Medium: Suitable for larger projects where processing time is a concern.

  • Low: Faster processing with less detail.

  • Lowest: Quickest processing, minimal detail.

Advanced Settings

Depth Filtering

  • Disabled: No depth filtering applied, can result in noisy point clouds.

  • Mild: Removes a small amount of noise, preserves more details.

  • Moderate: Balanced noise removal, suitable for most projects.

  • Aggressive: Removes a significant amount of noise, might lose some details.

Setting Up Batch Processing

In Agisoft Metashape, you can set up batch processing by following these general steps:

Open the Batch Process Dialog: Usually found under the "Workflow" menu, select "Batch Process."

Add Tasks: Use the dialog to add the desired tasks to the batch process. Common tasks include aligning photos, building a dense cloud, building a mesh, building texture, and generating a DEM or orthomosaic.

Configure Settings: For each task, you can specify the settings and parameters that should be used. These settings might include accuracy, quality, source data, and other specific options relevant to each processing step.

Order of Tasks: Arrange the tasks in the order they need to be executed. The software will perform these tasks sequentially.

Run Batch Process: Once all tasks are added and configured, start the batch process. The software will execute each task in the specified order, applying the settings consistently across all selected datasets.

Monitor Progress: You can monitor the progress of the batch process through the software's interface, which typically provides status updates and logs of the tasks being performed.

Export Settings 

When Export is clicked form the File dropdown, you come across a usual looking save file window. Input the desired file name and select FBX as the format option. Another window will pop up with more settings.

The Export Model - FBX window will have a bunch of settings. You’ll typically want to leave what is selected by default. The only difference being that the export texture should be in JPEG.

Vertex Colors: When enabled, the vertex colors will be included in the exported FBX file. This can be useful for preserving color information without needing a texture map.

Convert to 8-bit RGB: When enabled, the vertex colors are converted to 8-bit RGB format. This can reduce the file size and make it compatible with more applications, but might reduce color precision.

Vertex Normals: When enabled, the normals of the vertices are exported. Normals are important for rendering and shading, as they define how light interacts with the surface of the model.

Vertex Confidence: When enabled, vertex confidence values are included in the export. This can provide additional data about the reliability of each vertex, but is usually not necessary for most use cases.

Cameras: When enabled, the camera positions and settings used to capture the images will be included in the FBX file. This can be useful for reconstructing the scene in another application.

Export Texture: When enabled, textures associated with the model will be exported.

Options include:

JPEG: Compressed format, smaller file size, but lossy compression.

PNG: Lossless compression, larger file size, but retains quality.

TIFF: High-quality, uncompressed or optionally lossless compressed.

EXR: High dynamic range format, useful for detailed light and color information.

Include Comment: When enabled, you can include a comment in the exported FBX file. This can be "Generated with Agisoft Metashape" or something else.

Binary Encoding: When enabled, the FBX file will be encoded in binary format. This results in smaller file sizes and faster load times compared to ASCII format.

Precision: Defines the precision of the floating-point numbers in the export. Higher precision means more accurate data but larger file sizes.

Use UDIM Texture Layout: When enabled, textures will be organized using the UDIM (U-Dimension) layout, which is a method of arranging textures across multiple UV tiles. This is useful for managing large textures and detailed models.

Save Alpha Channel: When enabled, the alpha channel (transparency information) will be included in the exported texture. This is useful for models with transparency effects.

Embed Texture: When enabled, textures are embedded directly into the FBX file rather than being stored as separate files. This can make it easier to manage the model and textures together but can result in larger FBX file sizes.
