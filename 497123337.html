VARLab : Setting Up Bitbucket & Sourcetree

Created by Jordan Robinson, last modified by David Inglis on Sept 03, 2024

In this article, we’ll outline how to set up our Bitbucket account and Sourcetree to allow us to access and work on the different repositories we have at the VARLab.

At the VARLab we need to use SSH as our protocol as our Unity projects contain dependencies to other private repositories in the form of packages.

Since private repositories require some form of authentication, we need to set up our system to automatically allow Unity to authenticate us when it loads our project. This will allow us to access the private packages that a project depends on.

With all this in mind, we need to set up the Windows OpenSSH Client, generate our SSH Keys, install Git for Windows, install Sourcetree, and then set up our SSH connection with Bitbucket.

Installing OpenSSH Client

To access the VARLab’s private packages Unity will use your PC’s OpenSSH Agent and Git. If you do not have Git installed so that you can use it from the command line, you will need to install Git for Windows as well.

Below are expandable instructions on how to check your OpenSSH installation, install it if you don’t have it, and enable the service permanently on your PC.

Verify & Install OpenSSH Client

Windows 10

To see if your PC already has the OpenSSH Client installed, click on the Windows Icon, type in features and select Apps & Features.

From Apps & Features, select Optional Features and search for SSH under Installed Features to see if it’s already installed on your PC.

If the OpenSSH Client is not installed on your PC, click Add a feature, search for ssh, and add OpenSSH Client.

Once you have the OpenSSH Client installed, it is a good idea to restart your PC to ensure it’s all set up.

Windows 11

To see if your PC already has the OpenSSH Client installed, click on the Windows Icon, type in Add optional feature and select the option.

In this menu, search for OpenSSH Client in the Installed features to check if it’s already on your PC.

If the OpenSSH Client is not installed, open View features next to Add an optional feature and add the OpenSSH Client from there.

Once you have the OpenSSH Client installed, it is a good idea to restart your PC to ensure it’s all set up.

macOS

Check if SSH is Enabled

Open the Terminal App on Your MacBook, you can do this by searching “terminal” using the Spotlight search option of your computer or navigating through Applications > Utilities > Terminal.

Simply run and execute the following:

sudo systemsetup -getremotelogin

If SSH is on, you will get a message that reads “Remote Login: On”

You can now proceed to the “Generating SSH Keys” section of this document.

If SSH is off, you will need to follow these steps:

Set up Remote Login on your Mac

On your Mac, choose Apple menu  > System Settings, click General in the sidebar, then click Sharing on the right. (You may need to scroll down.)

Turn on Remote Login.

Open the Terminal and repeat the steps from the “Check if SSH is Enabled“ section. Your SSH should be on now and you can proceed to the “Generating SSH Keys” section of this document.

Enabling OpenSSH Service

To make sure your OpenSSH service automatically starts itself when your PC boots, click the Windows icon and type in Services.

Scroll down until you see OpenSSH Authentication Agent. If this service is not set to Automatic and is not running, right-click it and open Properties. In properties, you can set the Startup type to Automatic. After doing so, right-click on the service again and click Start.

Installing Git for Windows

Before installing Sourcetree, we will download and install Git for Windows. When you get to this page, change the setting to Use external OpenSSH.

The rest of the settings can be set however you’d like.

Generating SSH Keys

Generating SSH Keys from the Command Line

Open up the command prompt by using Windows Key + R and typing in cmd. Or click the Windows icon and search for cmd.

Make sure to open as an administrator, if you’re searching from the start menu you can Shift+Click on the result to do so.

Once you’re in the command prompt, type in ssh-keygen to generate your SSH key pair. For every prompt make sure you do not type anything at all and just hit Enter until it generates your keys.

If you include a passcode for your keys, Unity will not be able to use them as they would require you to input a password manually each time.

You will now have an SSH key pair located in C:/Users/YourName/.ssh where id_rsa is your private key and id_rsa.pub is your public key.

Private Key

While in the command prompt, type in ssh-add followed by the path to your id_rsa file such as ssh-add "C:/Users/YourName/.ssh/id_rsa"

The double quotes around the file path are important here as the command will not work for users with spaces in their name otherwise.

Public Key

Open up the folder you have with id_rsa.pub in it and open the file in Notepad or any other text editor. Copy the contents of that file and open up Bitbucket.

Once you’re in Bitbucket, click your icon and go to Personal Settings → SSH Keys and paste in your public key.

With that complete, return to the command prompt and type in ssh -T git@bitbucket.org to connect to Bitbucket via SSH and store your authentication for later use.

When prompted about the authenticity of the connection, answer yes to all prompts.

Close Unity & Unity Hub

After setting up SSH, it’s possible your authentication hasn’t carried over into Unity Hub if it’s already running. Since Unity launches with whatever credentials are present in the Unity Hub, you’ll need to exit that as well.

Setting Up Sourcetree

Follow the instructions in the section below for help installing Sourcetree.

Installing Sourcetree

  1. Download and run the installer for the Sourcetree desktop application.

  2. On the Registration page, select “Bitbucket” to launch your browser and complete the authentication. If you are not already signed in, you will be prompted to enter your email and password. Return to the installer when your browser indicates that “Sourcetree has been successfully authenticated.” If you skip this step, you will still need to sign in before you can clone the repository.

  3. On the Install tools page, uncheck the box for Mercurial, as it is not a tool we use at VARLab. There should be no checkbox visible for Git since it was installed in a previous step.

  4. On the Preferences page, the “Author name” and “Author email” fields may be automatically filled with your account information. If not, enter your information before completing the installation.

  5. If asked to provide an SSH Key, select No or simply close the dialogue.

Setting Up Git & SSH in Sourcetree

Now that Sourcetree is installed, we need to set it up to use Git for Windows and access our generated SSH Keys from earlier.

In Sourcetree, open up Tools → Options and under General, you can set the SSH Client to OpenSSH and navigate to the id_rsa private key file you generated earlier.

Afterwards, go to Tools → Options → Git and set Git Version to System.

With this all set up your Sourcetree is now ready to access all the VARLab Bitbucket repositories that you have permissions for.

A bug with Soucretree may cause the error message below to be displayed when opening the app.

If you see this error, try using Embedded Git instead and also Update Embedded before continuing.

This may resolve the issue, but if not, the error will not cause any problems with using Sourcetree for your work.

Cloning a Repository

In this section, we’ll clone the https://bitbucket.org/VARLab/unity-adventure-game/src/main/ repository as an example, and these steps can be repeated for other repositories as needed. Follow the link provided and click on Clone. Make sure the protocol is set to SSH and click Clone in Sourcetree.

If that doesn’t work, you can copy the URL to your clipboard and manually add the repository in Sourcetree by going to File → Clone / New and pasting it into the Source Path / URL field.

Choose where on your PC you want the project to be stored, then click on the Clone button and you’ll start downloading the repository. For large repositories, this process can take more than ten minutes.

Troubleshooting

Remote Host Identification

If you are receiving the warning that REMOTE HOST IDENTIFICATION HAS CHANGED, it is likely that you need to update your Bitbucket SSH keys on your machine.

Follow the Bitbucket blog post on resetting these keys.
