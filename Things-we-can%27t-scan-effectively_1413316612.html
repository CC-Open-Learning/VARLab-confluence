VARLab : Things we can't scan effectively

Created by estronge, last modified on Oct 24, 2024

As this project advances and we are getting more requests on things to scan and more knowledge on which subjects produce the best outputs, we thought it would be a good idea to take note of some traits in subjects that we tend to struggle with to ensure that various subjects are sent to the proper channels

If a subject is:

Transparent

and matches at least one of the following conditions:

  • Has few distinct features (could mean simple shape, no labels, flat sides, etc..)

  • Is not disposable + needs to be returned (if we can’t put markers/use transparent spray on it)

Then it is likely that we will be unable to produce anything that would be usable for a sim. Maybe as a placeholder, but even then, the result is oftentimes unrecognizable, unable to be optimized or both.

Reflective / Shiny

  • We can likely get a moderate quality scan using one of the handheld 3D scanners, but the environment + scanner may appear in the reflection of the texture, depending on how reflective it is. This can be remedied by applying a material in Unity, but if there are important details on the subject that can’t be lost, then it shouldn’t be scanned

Featureless

Meaning that there are zero or few distinct geometric/texture details on the subject, it will likely be difficult to scan using photogrammetry. We can probably get something usable with a 3D scanner but note that each condition that the subject matches in the following list will decrease the likelihood of a quality output:

  • Flat on all sides

  • No labels

  • Crevices/Sections that are only visible from a low angle

  • Is not disposable + needs to be returned (if we can’t put markers/use transparent spray on it)

  • Transparent

Check if subject can be purchased or modeled first if they fit these descriptions
