VARLab : Lighting In Unity

Created by Ian Kennaley, last modified on Jan 26, 2025

Article In-Draft

Purpose of this document is to cover a general workflow to setting up lighting in Unity. This document isn’t meant to be all-inclusive and some troubleshooting through self-research is to be expected. That said, this article should cover most of the basics of setting up lighting in Unity that you’ll encounter.

  • Step 1: Ensure that the Generate Lightmap UV’s option is enabled on your model imports
  • Step 2: Create new folder in an appropriate location and name it “Lighting” or something similar, and create a new Light Settings entry and name it “Main Scene Lighting” or something similar
  • Step 3: Open the Lighting Panel, Dock It, and then configure your settings as-needed
  • Step 4: Place your lights in your scene, configure them, and ensure that lights are set to “Bake” mode appropriately
  • Step 5: Add Light Probe Group and configure accordingly to your space
  • Step 6: Run your Lighting Bake
  • Additional Notes I: On Optimization

Additional Notes I: On Optimization

  • Enabling RealTime Light baking will result in a heavily bloated Lighting Data Asset. Enabling this feature will likely result in a Lighting Data Asset file size being in the 10’s - 100+ range, while leaving this feature disabled will result in a file size being in the 100kb+ - low 10’s mb range.

    The reason this feature causes such a large file size is that it will bake out a separate set of lightmaps for the realtime lighting in the scene. This is generally not necessary and is a very expensive (file size wise) feature to use as a result.

    • EX: In the Warehouse Scene used as the subject for this document, the RealTime Light Baking feature was used and this resulted in a Lighting Data Asset file size of 180mb. Disabling this feature resulted in a Lighting Data Asset file size of 173kb.
