VARLab : Agisoft Process Outline

Created by estronge, last modified on Aug 14, 2024

This page is being continually worked on as we find other aspects of the process that are worth noting. However, all currently existing sections are complete, to my knowledge.

  • Align Photos, Object, Boundary
  • Build Point Cloud

Add Photos (Folder)

  1. Navigate to Workflow > Add Folder

  1. Select the folder containing your image set

Align Photos, Object, Boundary

1. Navigate to Workflow > Align Photos

2. Change settings if needed, otherwise click OK

3. Wait… If successful, ignore steps 6 and 7

4. Correcting Object Transform properties

IMPORTANT

If you aren’t confident aligning the object and region, I recommend that you save a copy of your project at this point, as you can’t undo a rotation action!

Hopefully that will make it a bit less frustrating as I understand the gizmo sphere can be a bit finnicky when you are starting out :)

4.1 Identify Problem

Oftentimes, the transform properties of the scanned asset will be completely off.

These steps (any 4.x) should not be skipped.

Start by ensuring that numlock is on (however that works for your setup). Then press numpad7 to see what the top view is currently. If it is anything other than a view of the top of the object, you will need to rotate it.

4.2. Rotate Object

Make sure that you ignore the region around the object as it will be very out of sync, focus on making sure that the top of your object is facing the camera.

To confirm correct rotation:

(These actions should be repeated each time after rotating either the object or the region)

  • numpad7 (top)

  • numpad1 (front)

  • numpad 3 (side)

4.3 Rotate Region

For fine rotation controls, click and drag outside of boundary.

5. Resize Region (This will make model generation faster by removing excess geometry, also.. ignore the fact that the subject is different )

6. (Optional/Error) If you got this popup after completing the Align Images process, search for images without a checkmark beside them and..

7. Right click on non-aligned image > Remove Images

Build Point Cloud

This part of the process usually takes the longest, and it can typically be skipped. I have read that it is most beneficial for drone photogrammetry, and it hasn’t provided any notable differences in scan quality since we started incorporating it into our process.

1. Navigate to Workflow >Build Point Cloud

2. Change settings if needed, most likely they will be ok. Higher quality point clouds take exponentially longer, so keep that in mind!

3. Wait..

4. Probably still waiting…

5. I told you this process takes a while!!!

6. (Optional) Cancel because you are getting impatient

Build Model

1. Navigate to Workflow > Build Model

2. Change settings if needed, generally it is best to use default settings for either first or second pass (Makes mesh optimization much easier since there is more to work with). If you are concerned about your data set, running a low quality model build first can help to more quickly identify issues.

Trimming Unnecessary Geometry

Note: This step can be done before or after building the model. A similar effect can also be achieved using masks, but that will be documented in a different section. The primary reason this needs to be done is so that when you reach the mesh optimization step, there aren’t any faces being wasted on capturing the turntable, background, etc..

1. Navigate camera to good angle to see all geometry from scan

2. Choose a trimming tool (Rectangle or free form tend to work best for this purpose)

3. Trim as much unnecessary geometry as possible

4. Confirm that there are no holes in the mesh before moving to next step

Build Texture

1. Navigate to Workflow > Build Texture

2. Change any settings needed. Ideally, texture size should be 512 x 512 or 1024 x 1024.

Mesh Optimization

IMPORTANT

Save a copy of your project at this point, as you can’t undo a decimate action!

Then if the model loses too much detail or creates holes, you can reload.

1. Navigate to Tools > Model > Decimate Model

2. Change settings if necessary, refer to Ideal Output Parameters to find a guideline for the asset you are generating. (Start at the high face count target & move lower to test if geometry can be still be captured effectively)

3. Don’t panic if your texture became blurry! Build your texture the same way as before.

4. You will likely notice that the model looks just as good, with much better topology. If that is not the case, reload and adjust your target face count and try decimating again.

5. Second decimation pass. Low end of recommended target face count (800)

6. Third decimation pass (Halved one more time, down to 400 faces, AND I could probably do it again) This is an exaggerated example of the optimization process simply to show that these sorts of simple geometry objects can be cut down an insane amount through decimation. There is no expectation that a model should be decimated this many times.

7. JUST A NOTE, NOT PART OF ESSENTIAL PROCESS: This step is intended to show the effect of trying to decimate too far (100 faces). However, this could be used to generate LODS.

Export to Unity

1. Navigate to File > Export > Export Model

2. Save as .FBX in your designated subject folder (wherever you saved image set and project file)

3. Change settings if needed; most likely the following will be optimal.

4. Open Unity > right click in project window > Import new asset

5. Import both the texture and the model
