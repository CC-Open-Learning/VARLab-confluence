VARLab : Solution Implemented for Cross Team Testing Automatic Assignment

Created by Neethu Baby, last modified on Apr 03, 2025

The automatic assignment has been successfully executed as per the plan using a Power Automate flow and SharePoint lists. We chose Power Automate and SharePoint for several reasons. Firstly, Power Automate is a low-code solution that simplifies maintenance and reduces complexity, making it easy to manage without requiring extensive development efforts. Additionally, it doesn’t require deployment, which streamlines the process.

SharePoint was selected as the data source because it allows for easy transfer of ownership, is accessible to members, and ensures seamless collaboration across teams. Furthermore, SharePoint's integration with Power Automate provides a powerful, flexible, and secure environment for managing assignments and workflow automation efficiently.

Implementation Approach

The process begins with creating a SharePoint list, "CTT Testers," which is used to maintain all tester details. This list includes columns for tester name, primary project team, assigned project history, availability check, and assigned status.

Next is the "CTT Projects" list, which contains a comprehensive list of all projects. Project team leaders are responsible for updating this list before the Cross Team Testing to specify how many testers are required for their project. Additionally, if a project is not scheduled for testing during the current sprint, team leaders can indicate this in the list as well.

Lastly, the "CTT Assignments" list will contain the results of the automatic assignment process. The Power Automate workflow will automatically delete and add new entries to the assignment list each time it runs. This list includes the tester name and the project assigned for testing. The list is grouped based on the primary project team to ensure better visibility and organization.
