VARLab is beginning to use Connecter as a platform for sharing, versioning, and workflow on our art assets. This will make it easy for developers and artists to find and re-use assets across projects, as well as seeing progress through workflow statuses.

The Connecter application previews assets from a file share that we operate, but uses online accounts through Connecter Server to orchestrate collaboration, workflow, team roles, and access control. The file share is stored in our Azure Cloud services and we need VPN software to ensure secure access from remote networks.

Components

Connecter - workstation software that previews 3D models and more

Connecter Server - an online service that manages the Connecter application’s user accounts

  • concurrent access between team members

  • access rights to content within the file share depending on folder, team, role, etc.

Azure File Share - storage for all of our art, hosted in Conestoga’s cloud account

Azure VPN Client - a networking tool for secure access to the Azure File Share from remote locations

  • any VPN client software that can understand the configuration file could work, this needs testing

Procedure

Set up on-campus access to File Share

Open terminal and run the attached script file to mount the connecter-asset-store share to Z: on your computer. This script will only run successfully and your Z: drive will only be available while connected to the campus network.

Verification

The script should complete without error messages, and you should be able to see files in the Z: drive through File Explorer (only while connected to campus network)

Install and Configure VPN Client, and Mount File Share

Connecting to Azure Storage Account "varlabsazurestorage" via VPN - VARLab - Confluence (atlassian.net)

It is important to note that the VPN Client only connects successfully from outside of the Conestoga network.

VLT Developers will be onboarded to Bitwarden (this requires an invite from an admin) and use those instructions for access to the file share credentials. Other developers should have the file share configured by ITOps or the VLT Developer on their team.

Remember to mount the files share to a different drive letter (instructions will expect it is Y:)

When working remotely, open your VPN client and connect to P2S_vnet so that you can see contents in Y:

Verification

The Azure VPN Client application connects successfully (only from a non-campus network)

The contents of Y: are visible (only with VPN client connected)

Invitation to Connecter Server Workspace

You will be sent an email when you are invited to the Conestoga VARLab Connecter Workspace. Please click the link to Connecter.

Install Connecter and Import Shared Asset Folder

Log in to https://teamwork.connecterapp.com/ and click the top right menu icon on the page to see the download link.

Download and install the software

Run Connecter on your workstation and log in using the menu on the top right of the title bar

Add the mounted network file share by clicking the + icon in the Folders pane header.

When switching between on-campus and remote work you will have to re-locate your asset folder from Z: to Y: because the network share connections are not mutually compatible with VPN and local access.

Verification

The asset storage contents should be visible in the Connecter application (on- or off-campus depending on appropriate use of Y: or Z: drive)
