Step-by-Step Guide for Document Maintenance

  1. Create a Confluence Page for Each Application

  • Ensure that each application has a dedicated Confluence page outlining the steps to follow.

  • Currently, a Confluence page exists only for the Climate Change application, which can be accessed [here].

  1. Keep Documents Updated

  • Regularly review and update the Confluence pages to reflect the latest information and steps.

  • Include new workflows, testing steps, and any changes in process promptly.

Step-by-Step Guide for Issue Verification Using MS Forms

Documenting Fixed Defects from Third-Party Vendors

  • Add fixed defects into the MS Form to enable testers to verify their availability during testing.

Create and Configure the MS Form

Step 1: Start a New Form

  • Add a question for testers to confirm the availability of the defect. Use a choice type question with options: Yes or No.

Step 2 : Add Branching

  1. If the answer is Yes, prompt the user to:

  • Select the issue type.

  • Provide an explanation.

  • Enter the defect ID.

2. If the answer is No, proceed to the next question.

Step 3: Define Testing Period

  1. Set the start date, end date, and time for the verification process.

Step 4: Share the Link with the Team

  1. Provide the MS Form link to all relevant team members to begin the verification process.
