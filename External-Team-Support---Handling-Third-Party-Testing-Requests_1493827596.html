Overview

This page outlines the process for requesting external third-party testing, handling fixed defect lists, and recording results.

Process Workflow ( Before the improvement )

Receiving the External Request

  • A new build will be provided for testing from the vendor along with the fixed defect list.

Forming the Testing Teams and Sharing the Test Request

  • The fixed defect list will be posted in the designated Teams Chat, requesting the testers to review the list and verify the fixes against the new build.

Playing the Simulation & Defect Verification

  • Testers will check if the so called ‘Fixed Defects’ are actually fixed or not. If not fixed or found new defects during the testing , those should be recorded in the excel sheet provided by Cristina Barillas .

Process Workflow ( After the improvement )

New workflow has been introduced to ensure that the simulation's testing process is strengthened, to maintain higher quality and reliability of results.

As per the teams structure, testing will be conducted accordingly. During each testing session, a dedicated Teams chat will be available to facilitate communication and address any concerns in real-time. A set of comprehensive steps will be shared with all participants, including the following resources:

  1. DLX Link – Access to the testing environment.

  2. Confluence Page Steps – Detailed instructions for the testing process.

  3. Issue Verification – Submission form via Microsoft Forms for reporting and validating issues.

  4. Bug Reporting – SharePoint list for logging identified bugs.

Instructions for Testers:
Each tester is required to follow the outlined steps diligently to ensure the testing process is completed efficiently and thoroughly.
