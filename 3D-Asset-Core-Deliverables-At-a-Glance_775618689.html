VARLab : 3D Asset Core Deliverables At a Glance

Created by Topher Rouleau, last modified on Jul 02, 2024

This document was updated on Sept 27th, 2023 to reflect changes in the expected file structure and make it easier to blow away source files

This document serves as a quick reference of what should be delivered for a finished asset. It should be a final check to make sure that all needed file are being delivered, and that the files are somewhere that will be expected.

Delivery Location

All files should currently be delivered to the VAR Lab’s SharePoint, and then into the Art Assets folder for that project.

At this point in time the files are being delivered to specific project folder, though the vConestoga team is currently working on an internal asset library. Exciting!

As an example, if the model you are making is going to be implemented into Heavy Equipment - Bulldozer, the file the Art Assets folder in the Heavy Equipment - Bulldozer folder.

Some assets may not have been built for a specific project in mind. If this is the case, we can begin setting up a Common Assets folder on SharePoint, though one does not exists at this time

In the event that you have generated an asset but do not have access for to SharePoint (perhaps you are a Work Placement Student), the files can be zipped and sent to Ian or Topher and one of them will give the file it’s home

File Structure

Submitted files should have a consistent structure, and our approach is to structure the files so that only one folder needs to be downloaded to retrieve all of the related files. This approach will mean that folders will have to be moved once imported into Unity, and other files or folders can be deleted entirely.

The current proposed file structure is as follows. Note that any line in bold should be in every completed asset submission, and anything that are in italic are other files that you may have, but won't always:
