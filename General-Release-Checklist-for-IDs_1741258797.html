Created by Katrina Fraser on Apr 09, 2025

eConestoga DLX Course Shell To-do’s

        Course shell needs to follow the naming convention for the VARLab course shells, so should be VARLAB – [Project Name]

        If the shell is named anything other than that, ID/PI needs to contact eConestoga helpdesk with a name change request.

        Confirm with Russ what the approved name is.

        Confirm that all documentation in the shell shows the approved project name, and not any other names that were used in the development process. For example in TE, rename everything TE to the approved project name Conestoga Trades: Electrical Simulator, (and take the TE label off everything).

        Any files in the shell should be stored in the Manage Files section of the eConestoga shell, and use quick-links to them. This may mean uploading those files, and changing the file links to a quick- linking internal practice.

What should be in the eConestoga DLX course shell

Faculty Resources, including

        Introduction to the DLX

        Instructional Power Point slides

        Navigation guide/Tutorial Instructions

        Navigation/Tutorial Video

        Debrief Questions

        Instructions on how to import the DLX into a different shell

        Technical requirements (Release notes)

 

Student Resources, including

       Navigation guide/Tutorial Instructions

        Navigation/Tutorial video

        Survey

        Build Link

        Technology Tips (adapted to a user friendly format from release notes, 1-5 points max including)

        Device specs

        Browsers

        Multiple tabs info (if applicable)

        Don’t use browser back button (if applicable)

        Performance info (ie. plug in laptop for best functionality)

        Offline play (wait to get online before closing tab)

 

eConestoga Shell Last Pass Requirements: Make Sure:

        The project PI has instructor access in the shell

        Russ has instructor access in the shell

        The Lead Dev has instructor access in the shell

        Your CEs and other key instructors have access to the shell (or confirm the CEs can share it)
