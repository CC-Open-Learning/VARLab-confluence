VARLab : Discarding Changes From A Pushed Commit

Created by Jordan Robinson on Jul 05, 2022

If you have a situation where you accidentally push changes to a file you didn’t mean to push, we can do a soft reset to undo the commit and return everything back to unstaged changes.

First you’ll need to check out the branch you want to remove the change from in Sourcetree and go to Actions > Open In Terminal.

In the terminal, type in git reset HEAD~1 to undo the last commit locally and return all the changes to unstaged changes.

You can now discard the changes you didn’t mean to include in your commit and create a new commit.

Once you’ve created your new commit, you’ll notice you have commits to push and pull as your local branch is now out of sync with Bitbucket.

Since we know the local version of the branch is the true version, we can Push and select Force Push. If force push is not enabled, you will need to enable it in Tools > Options > Git > Enable force push first.

After force pushing, your new commit without the file you accidentally committed will be up on Bitbucket.
